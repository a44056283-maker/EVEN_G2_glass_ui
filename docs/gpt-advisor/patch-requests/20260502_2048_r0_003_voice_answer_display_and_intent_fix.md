# R0-003 当前补丁任务：呼叫天禄语音结果展示与意图路由修复

创建时间：2026-05-02 20:48

## 本轮目标

修复“呼叫天禄”当前真实问题：

1. 手机端按住录音不稳定，按住几秒会像自动断掉。
2. 回答仍出现占位话术，例如“收到，会把结果显示在这里 / 发送 Telegram”。
3. Voice Debug 显示在普通用户回答下面，影响使用。
4. 语音历史没有稳定保存最近 15 条。
5. 交易问题没有完全复用最新 7 项摘要 + MiniMax 评测。
6. “看一看/这是什么”等视觉意图没有稳定触发现有视觉识别流程。

## 本轮范围

只修改：

- 呼叫天禄语音控制器。
- 手机端按住说话交互。
- 眼镜端呼叫天禄录音入口。
- handleTranscript / routeVoiceIntent。
- 语音历史。
- 呼叫天禄结果展示。

不修改：

- R1 视觉相机状态机。
- 视觉识别基础上传逻辑。
- 交易写操作。
- OpenCLAW 网关配置。
- 整体 UI 大重构。

## 验收标准

- 手机端必须按住才录音，松手才停止。
- 录音期间显示时长和状态，不应几秒自动断。
- 眼镜端进入呼叫天禄后自动开始录音，R1 单击停止并识别。
- ASR 返回的 transcript 必须进入 handleTranscript。
- 普通问答不能再返回占位句。
- 交易意图必须调用 `/trading/overview` 并使用 `aiAssessment`。
- 视觉意图必须调用已有视觉结果或切换到现有视觉识别流程。
- 语音历史最多 15 条。
- Debug 信息只能在调试区域，不放在用户回答下方。
- 完成后打包 EHPK 并写测试报告。
