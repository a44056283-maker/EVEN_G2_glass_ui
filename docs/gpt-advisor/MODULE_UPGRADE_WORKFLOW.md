## 总原则

1. 每轮只升级一个模块。
2. 每轮先生成 `patch-requests/` 任务文件。
3. 每轮完成后生成 `test-reports/` 测试报告。
4. 每轮必须更新：
   - `CURRENT_STATUS.md`
   - `NEXT_ACTIONS.md`
   - `ISSUE_REGISTER.md`
   - `MODULE_MAP.md`
   - `TIMED_PROGRESS_LOG.md`
5. 交易系统第一阶段只读，禁止下单、平仓、改杠杆、改策略、启停机器人、提现。
6. G2 眼镜端不是 HTML/CSS，必须走 GlassRenderer / Even Hub container。
7. G2 原生麦克风主链路必须是 `bridge.audioControl(true)` + `event.audioEvent.audioPcm`。
8. 手机网页麦克风只能作为 fallback，不可污染 G2 主链路。

## 第一阶段执行定义

阶段 1 不是重新做全部语音模块，而是修当前真实问题：

```text
手机端：
按住圆形按钮 -> 真实录音 -> 松开结束 -> 本地 ASR -> handleTranscript -> 显示答案

眼镜端：
进入呼叫天禄 -> 自动开始 G2 PCM 录音 -> R1 单击停止 -> 本地 ASR -> handleTranscript -> 显示答案

通用：
答案直接显示在呼叫天禄结果卡片
不再显示“会发送 Telegram/显示在这里”这种占位话术
不再把 Voice Debug 放在普通用户回答下面
语音历史最多 15 条
交易问题必须使用 /trading/overview 的 7 项摘要 + MiniMax 评测
视觉问题必须触发现有视觉识别或最近视觉结果
```

## 阶段 1 验收
