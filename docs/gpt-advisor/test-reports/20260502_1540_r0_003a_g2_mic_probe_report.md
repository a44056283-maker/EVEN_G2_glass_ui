## 当前点击路径

手机网页或 R1 选中 `呼叫天禄 / 语音对话` 后：

```text
voice-button / voice-record-action
→ enterVoicePage()
→ runVoiceFlow()
→ runG2MicWebSocketVoiceFlow('assistant')
→ startGlassMicProbe()
→ WebSocket /audio?mode=probe&source=g2
→ bridge.audioControl(true)
→ bridge.onEvenHubEvent(audioEvent.audioPcm)
```

## 已修复点

1. 进入“呼叫天禄”后直接启动 G2 Mic Probe。
2. G2 主链路不再默认请求手机/耳机麦克风。
3. 未检测到 Even Bridge 时，只提示使用 Even App 真机或文字输入，不再自动调用 `getUserMedia({ audio:true })`。
4. `startGlassMicProbe()` 增加 Voice Debug 回调。
5. `audioEvent.audioPcm` 转 `Uint8Array` 的兼容处理已增强，支持：
   - `Uint8Array`
   - `ArrayBuffer`
   - `number[]`
   - `ArrayBufferView`
   - `ArrayLike<number>`
6. 手机网页新增 Voice Debug 面板。
7. 后端 `/audio?mode=probe` 返回 `mode/source/totalBytes/chunks/lastChunkBytes`。
8. 新增 `apps/evenhub-plugin/src/voice/g2MicProbe.ts` 作为语音模块路径入口，复用当前 Glass Mic Probe 实现。

## ASR 状态

本轮不修 ASR。

当前 R0-003a 只验证：

```text
G2 麦克风 PCM 是否到达插件
PCM 是否发到后端 /audio?mode=probe
```

即使 ASR 未配置，也不应影响 probe。

- `audioControlCalled: true`
- `wsStatus: open`
- `totalBytes > 0`
- `chunks > 0`

```text

```
