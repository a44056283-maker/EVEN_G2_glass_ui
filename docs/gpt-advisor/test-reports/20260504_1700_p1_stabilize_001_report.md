## 代码审计结果

### 1. TTS 来源审计

**文件**：`speech.ts:86-104`、`minimax-adapter/src/index.ts:82-137`

**链路**：
```
speakIfEnabled() → requestTts() → /tts → synthesizeMiniMaxTts()
                                            ↓
                         audioBase64 存在 → playAudioDataUrl()
                                              ↓ 成功 → { method: 'minimax-tts' }
                                              ↓ 失败 → speakWithBrowser()
                         audioBase64 不存在 → speakWithBrowser()
                                              ↓ { method: 'browser-fallback' }
```

**voiceId 默认值**：`female-shaonv`（config.ts:11）

**确认结果**：
- 代码链路正确：MiniMax TTS 优先，失败后 fallback
- **需要真机确认**：设置页应显示 `played.method` 为 `minimax-tts` 或 `browser-fallback`
- `.env` 的 `MINIMAX_API_KEY=replace_with_your_key` 是占位符；若服务器环境使用 `.env.example` 的真实 key，则走 MiniMax TTS；否则走 browser fallback

**真机验证方法**：
1. 重启 API Server（使 `.env` 配置生效）
2. 进入设置页，点击"试听当前声音"
3. 查看设置状态文字：`试听成功：MiniMax TTS 音频` = MiniMax TTS，`已退回系统朗读` = browser fallback
