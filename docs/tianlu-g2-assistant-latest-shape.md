# 天禄 G2 助手最新形态基线

来源：`视觉语音+天禄助手.pages`

整理日期：2026-05-01

## 一句话定位

把当前软件从“G2 视觉语音助手”升级为：

```text
天禄 G2 助手
```

它不是只做拍照识别，而是通过 G2 眼镜、手机、MiniMax、G2 Bridge、OpenCLAW 和交易系统只读工具，形成一个随身语音助手。

## 用户自然交互

用户打开 Even 软件后，可以直接说：

```text
你好天禄，帮我看一下我们的交易机器人运行如何。
```

系统应该完成：

1. G2 麦克风听到语音。
2. 检测唤醒词：`你好天禄` / `天禄`。
3. 录下后面的指令。
4. ASR 转文字。
5. G2 Bridge 后端把问题交给 OpenCLAW。
6. OpenCLAW 调用交易机器人状态工具。
7. MiniMax / OpenCLAW 生成简短结论。
8. G2 显示结论。
9. 手机或蓝牙耳机用女声朗读。

## 推荐最终架构

```text
Even G2 插件
├─ 打开 G2 麦克风
├─ 识别唤醒词“你好天禄”
├─ 采集指令音频
├─ 显示“天禄正在检查……”
└─ 连接 G2 Bridge 后端
        ↓

G2 Bridge 后端
├─ ASR：语音转文字
├─ 会话管理
├─ 调 OpenCLAW
├─ 调 MiniMax TTS
└─ 返回短答案 + 播报音频
        ↓

OpenCLAW
├─ 运行时大脑
├─ 调用交易机器人状态工具
├─ 调用白名单价格工具
├─ 调用持仓 / 盈亏 / 风险工具
└─ 生成只读结论
        ↓

交易系统只读工具
├─ 白名单实时价格
├─ 所有交易机器人状态
├─ 持仓和盈亏比
├─ 风控 / 自动驾驶状态
└─ 不下单、不平仓、不改仓
```

## 角色分工

### OpenCLAW

OpenCLAW 是运行时大脑。

负责：

1. 理解用户意图。
2. 调用交易系统只读工具。
3. 汇总交易机器人状态。
4. 生成简短可靠的结论。
5. 遵守 V6.5/V6.6 和只读边界。

### Codex

Codex 是开发和维护工具，不是运行时交易执行者。

负责：

1. 写代码。
2. 改代码。
3. 调试 G2 插件和后端。
4. 维护文档。
5. 打包 EHPK。
6. 帮用户推进版本。

### G2 插件

G2 插件是随身入口。

负责：

1. 显示状态。
2. 接收 G2 点击 / R1 戒指操作。
3. 调用手机摄像头。
4. 调用 G2 麦克风。
5. 把结果显示到镜片。

## 当前缺口

### 1. ASR 仍未真正打通

当前 G2 麦克风已经能收到 PCM 音频，但 MiniMax 公开 OpenAPI 暂未看到稳定 ASR 端点。

下一步需要二选一：

1. 找到 MiniMax 账号可用的 ASR / Realtime endpoint。
2. 临时接入独立 ASR 服务作为后端 adapter。

### 2. OpenCLAW 运行时需要继续联调

当前 `/ask` 已接 OpenCLAW 代理，但公网入口后方网关仍可能返回 502。

下一步需要确认 OpenCLAW 后端网关稳定，并逐步新增：

```text
/openclaw/tools/trading-overview
/openclaw/tools/bot-status
/openclaw/tools/positions
/openclaw/tools/prices
```

### 3. 交易系统仍是记忆只读模式

当前交易面板只读检索 `data/remote-memory-cache`。

下一步要接：

1. `9099` 聚合控制台。
2. `7891` 风控 / 事件总线。
3. 白名单实时价格。
4. 机器人持仓和盈亏只读接口。

### 4. R1 戒指控制还需要细化

当前逻辑：

1. 默认点击：拍照识别。
2. 交易面板打开后点击：刷新交易概览。

后续建议：

1. 单击：确认 / 刷新。
2. 双击：返回主页面。
3. 长按：呼叫天禄 / 停止朗读。

3. G2 只读，不改仓。
