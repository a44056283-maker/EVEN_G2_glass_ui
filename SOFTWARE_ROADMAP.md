### 阶段 2：网页视觉识别闭环

状态：下一步优先

目标：

先不碰 R1，先把网页上最基础的视觉识别跑通：

```text
视觉识别 -> 拍照/选图 -> /vision -> 显示结果 -> 朗读/历史记录
```

任务：

- [ ] 点击一级“视觉识别”只进入视觉页，不自动拍照。
- [ ] 二级“拍照识别”能打开手机相机或文件选择。
- [ ] 拍照/选图后压缩图片。
- [ ] 调用 `POST /vision`。
- [ ] 网页显示识别结果。
- [ ] G2 眼镜显示 `reply` 结果。
- [ ] TTS 可用时朗读，不可用时不影响文字显示。
- [ ] 写入历史记录。

验收标准：

- 手机上能完成一次真实拍照识别。
- `/vision` 返回不报错。
- 结果能显示在网页。
- 如果眼镜连接，结果能显示到眼镜。

### 阶段 6：G2 Mic Probe

状态：代码已有，真机待验收

目标：

只验证 G2 麦克风 PCM 是否到达，不接真实 ASR。

任务：

- [ ] `bridge.audioControl(true)` 开启。
- [ ] `audioEvent.audioPcm` 收到 bytes/chunks。
- [ ] WebSocket `/audio?mode=probe` 收到 PCM。
- [ ] 5 秒无 PCM 时显示明确诊断。

验收标准：

- 不出现浏览器麦克风 `not-allowed` 作为主链路错误。
