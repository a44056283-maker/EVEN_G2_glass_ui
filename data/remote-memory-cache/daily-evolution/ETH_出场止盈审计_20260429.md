## 结论

- 今天 ETH 确实出现“出山/天眼已经评测到止盈或全平候选，但没有下发执行”的情况。
- 主因不是 ETH 没有被扫描，而是 9099 出场执行链路存在动作口径不一致:
  - 出山AI/天眼AI可能输出 `全平出场`、`半仓出场`、`EXIT_FULL`、`EXIT_HALF`、`half_close`。
  - 底层执行函数原来只识别 `full_close`、`partial_close`、`giant_volume_reverse`。
  - DecisionEngine 返回 `half_close` 后，底层 `_execute_exit_ai_decision()` 不识别，容易落入 `no_auto_execute`。
- 第二个主因是 V6.5 P2 止盈被 `near_sr` 硬门控拦住:
  - 5x 时 P1=7.5%，P2=12.5%，P3=17.5%。
  - P2 对应 50% 止盈，本应不被 `near_sr=False` 完全拦截。
  - 今天 ETH 在 9092 端口一度达到 12.89%，已经超过 5x P2=12.5%，但 DecisionEngine 仍给了观望。

  - 5x 时 P2=12.5%，达到后执行 50% 止盈，不再被 `near_sr=False` 拦住。
- DecisionEngine 增加动态止盈全平保护:
  - 出山AI明确 `full_close` / `EXIT_FULL`，或动态止盈总分低于20且盈利，置信度>=70时，允许进入全平执行链路。
