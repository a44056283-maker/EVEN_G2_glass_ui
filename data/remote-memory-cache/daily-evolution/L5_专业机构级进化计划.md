# L5 专业机构级交易系统进化计划

生成时间: 2026-04-28

## 目标

把当前“天眼AI + 出山AI + M1-M4 + 12机器人执行器”的 L3.5 系统，升级为具备可回放、可验证、可归因、可控自进化、组合风控能力的 L5 交易系统。

## 阶段规划

### 阶段1: 稳定性与数据完整性

- 建立 L5 市场微结构数据层: 订单薄、OI、爆仓、贪婪指数。
- 所有 M1-M5 数据必须有 source、ts、validation_level、fallback_reason。
- 所有新增数据模块统一采用三级缓存: L1内存热缓存 -> L2本地JSON/SQLite -> L3实时数据源；实时源失败必须回退本地缓存并标记降级原因。
- 9099 重启后自动恢复机器人注册、缓存状态、自动驾驶开关状态。

### 阶段2: 回测与订单薄补齐

- 回测样本必须绑定当时 M1-M5 快照。
- 每次天眼入场、出山出场都记录订单薄倾斜、点差、OI、爆仓、情绪。
- 回测与实盘使用同一套参数注册表，禁止出现“回测一套、实盘一套”。

### 阶段3: 参数实验台

- V6.5 参数进入统一 registry。
- 支持 walk-forward、分币种、分市场状态、分交易所参数对照。
- AI只能生成候选参数，必须经过回测/前测/灰度后才能生效。

### 阶段4: 组合风控

- 12机器人统一净敞口、保证金占用、同向拥挤、相关性风控。
- 增加单日最大亏损、连续亏损、交易所异常、数据源降级熔断。
- 把 `stoploss=-999` 替换为灾难级硬止损/账户熔断护栏。

### 阶段5: 机构级执行

- 建立订单生命周期: 信号、授权、下发、挂单、成交、部分成交、撤单、滑点、拒单、重试、对账。
- 记录每个机器人和每个交易所的执行质量。
- 低流动性、点差异常、订单薄倾斜反向时禁止追单。

### 阶段6: 可控自进化

- 天眼AI/出山AI每天读取日志，输出错误归因、参数建议、新场景建议。
- 自进化结果进入候选池，不直接改实盘参数。
- 每日生成进化日志，记录完成项、失败项、下一步。

## 风控原则

L5可以增强判断，但不能绕过天眼/出山唯一执行权。M1-M5负责采集和评分，天眼AI负责入场授权，出山AI负责出场授权，机器人只做执行。

## 系统保命护栏

进化过程的第一前提是不能把电脑、9099控制台、交易机器人和数据源进程搞死机。所有进化任务必须遵守以下硬规则:

- 先保交易系统运行，再做进化任务；任何进化任务不得抢占交易机器人、9099控制台、天眼AI、出山AI的核心资源。
- 定时进化任务默认只读审计，不自动大规模扫描全盘，不自动复制大文件，不自动压缩备份，不自动清理交易核心文件。
- 禁止在交易时段执行高CPU/高内存/高磁盘I/O任务；回测、全量日志扫描、全量样本重建必须限时、限并发、可中断。
- 外挂硬盘只允许做低优先级缓存和备份；当外挂盘响应慢、掉盘或I/O等待升高时，立即降级到本地24小时缓存，不阻塞9099和交易机器人。
- 每个定时任务必须有超时上限、接口延时记录和失败降级；单个接口不能无限等待。
- 禁止定时任务自动重启9099、9090-9097、8081-8084交易核心服务；除非确认服务不可用且不会造成正在交易的机器人重复下单。
- L5回测与样本重建必须分批处理，不能一次性加载全部历史数据进内存。
- 发现内存、磁盘、I/O等待异常时，进化任务立即停止，只记录问题到每日进化日志。

## 新增模块硬规则

以后所有新增数据模块必须默认三级缓存加载，禁止只做单源实时请求。任何无法实时获取的数据不能显示裸 `--` 或空值，必须显示真实值、计算中、无触发或数据降级来源。
