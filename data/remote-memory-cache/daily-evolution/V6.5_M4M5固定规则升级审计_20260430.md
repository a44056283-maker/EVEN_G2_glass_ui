## 结论

当前机器人实际执行的 V6.5 入场硬规则仍以 M1-M3/M2 S/R 为主，M4-M5 已经进入 9099 天眼AI/出山AI提示词、英雄卡和评分话术，但还没有作为机器人侧固定硬规则完整闭环。把 M4-M5 做成固定规则有提升精准捕捉的可能，最适合的定位不是扩大入场，而是:

- 过滤追高做多、追空做空。
- 识别 ATR 极端波动时的杠杆降档。
- 识别 OI减仓、盘口反向、点差过大、爆仓风险。
- 减少错误方向、过早出场和噪音下单。

但在进入实盘前必须先修一个数据质量问题: 当前 `l5_replay_samples` 中发现 M4样本交易对错配，部分 BNB 样本的 `m4_data.pair` 为 BTC/USDT。这会污染历史回测，不能直接用错配样本证明收益提升。

## 推荐固定规则边界

入场:

- V6.5 M1-M3 仍是先决条件，M4-M5不能绕过 M1/M2/SR。
- M4不能为了开仓主动放宽，只能确认或拦截。
- RSI>70 禁止追多，RSI<30 禁止追空。
- RSI极端 >75 或 <25 时禁止新开仓，除非进入单独的反转猎杀候选池，且仍需天眼AI许可。
- ATR>5% 只降杠杆，不直接否决。
- ATR>8% 禁止新开仓，避免高波动扫损。
- OI加仓 + M1同向 + M2有效，可提高置信度，不提高V6.5给出的最高杠杆。
- OI明显减仓时，不追趋势单。
- M5点差>=0.08%禁止追单。
- M5盘口反向超过12%禁止入场。
- M5单源/缺失不得加分，最多维持原判或降级。

出场:

- M4/M5不能替代 P2/P3 硬止盈。
- M4/M5适合保护利润、收紧止损、发现方向错误。
- 仅 RSI 偏离不能单独平仓，必须叠加 M1/M2/M3 或 PnL路径。
- 同向爆仓、反向盘口、OI减仓同时出现时，出山AI可进入半仓保护候选。

## 数据质量风险

本次审计发现:

- `l5_replay_samples` 当前样本数: 1391。
- M4非空样本: 1391。
- L5非空样本: 1391。
- 发现 M4交易对错配样本: 366。
- 典型现象: BNB/USDT 样本里 `m4_data.pair` 出现 BTC/USDT。

这说明 M4回填/样本物化缓存存在串币风险。正式把 M4写入硬规则前，必须先保证:

- 每条样本的 `pair == m4_data.pair`。
- M4回填缓存 key 必须按交易对隔离。
- 回测脚本必须剔除 M4错配样本。

## 预期效果

如果数据修正后回测支持，M4-M5固定规则预计能实现:

- 降低追高做多和追空做空。
- 降低点差大、盘口反向时的噪音下单。
- 在高ATR阶段自动降杠杆，减少扫损。
- 提高出山AI保护利润与方向纠错的一致性。
- 让天眼AI/出山AI从“提示词判断”升级为“遵守固定硬边界”。
