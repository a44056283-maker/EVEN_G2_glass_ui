# SOL 提前半仓平仓事故审计 - 2026-04-30

## 结论

本次 9093-9097、8081-8084 的 SOL 持仓提前半仓，不符合 V6.5 出场逻辑。真实订单表现为 `force_exit(amount=...)` 且数量连续按剩余仓位约 50% 衰减，主要指向机器人内部的 L5 动态止盈 / L2 时线止盈 / 部分平仓追踪链路，而不是天眼或出山 AI 在 9099 决策表里留下的直接执行记录。

## 证据

- 9093-9097 的 SOL 订单均出现多次 `sell`，数量形态接近 1/2、1/2、1/2 递减。
- 9095 的 SOL 在约 `-2.74%` 杠杆收益附近被 `force_exit` 平掉，未达到 5x 的 V6.5 P1 触发线 `7.5%`。
- 9099 的 `ai_exit_evaluation`、`ai_tianyan_intervention`、`alert_records` 未发现今日 SOL 相关半仓执行记录。
- Bot Agent 日志和本地 `exit_log` 未发现今日 SOL 的 `half_close` 命令记录。

## 发现的问题

1. L5 动态止盈的状态 key 只按 `port + pair` 隔离，未绑定 `trade_id`，存在新仓继承旧仓止盈档位的风险。
2. 部分平仓追踪把 Freqtrade 已经缩减后的 `trade.amount` 再减 `_partial_exit_tracker`，会造成重复按剩余仓位继续砍半的级联风险。
3. 9099 天眼执行入口原来只对 `EXIT_FULL/HUNT_REVERSE` 做硬闸，`EXIT_HALF` 仍有软条件绕过空间。
4. Bot Agent 接收 `half_close` 命令时缺少本地 V6.5 二次闸门。
5. L2 时线止盈属于止盈逻辑，但缺少“未达 P1 不得普通半仓”的发育保护。

## 已修复

- 动态止盈状态改为 `port + pair + trade_id`，避免新仓继承旧仓档位。
- L1/L2/L5 部分平仓统一使用当前实时剩余仓位，不再用累计已平数量二次扣减。
- 增加 V6.5 发育保护：未达 P1，且无结构破位 / 强平风险授权时，禁止普通半仓、分批止盈、动态止盈。
- 9099 天眼 / 出山执行入口将 `EXIT_HALF` 纳入 V6.5 硬闸。
- Bot Agent 的 `half_close` 命令入口增加本地 V6.5 闸门。
- 同步到本机 9090-9097、电脑B 8081-8084 的策略文件，并重启 12 个交易机器人和 12 个 Bot Agent。
