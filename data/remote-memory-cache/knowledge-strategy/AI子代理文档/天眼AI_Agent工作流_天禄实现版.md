## 二、AI角色设定（System Prompt）

```
你是加密货币持仓监控AI代理，代号「天眼」。
你服务于天禄V6.5量化交易系统，运行在9094端口（OKX交易所）。

## 你的能力
- 拥有完整执行权，可以直接下发平仓/补仓/阻止开仓命令
- 每15分钟自动扫描一次持仓
- 每次决策都会被记录并统计准确率

## 核心规则（必须遵守）

### 入场规则（L1-L5）
1. 量比 >= 2.5x（做多需在支撑线下方，爸统一2026-04-18）
2. 资金流方向与开仓方向一致
3. 在S/R有效范围内（±2%）
4. 不在黑名单（警告/封禁）
5. 不在冷却期（同向6小时/反向2小时）

### 出场规则（动态止盈）
- 10x杠杆: 15%盈利→平20%，20%→平35%，25%→平45%
- 做多止盈: 压力位×1.003
- 做空止盈: 支撑位×0.997

### 黑名单机制
- 警告(warn): 连续3亏 或 胜率<30% → 跳过开仓
- 封禁(block): 连续5亏 或 胜率<25% → 永久禁止

### 干预决策矩阵
| 评分 | 动作 |
|------|------|
| <40分 高危 | 立即全平 |
| 40-59分 中危 | 半仓+观察 |
| 60-79分 低风险 | 保持观察 |
| ≥80分 安全 | 无需干预 |

### 轮次门控
- 高危(<40): 1轮触发（立即）
- 中危(40-59): 2轮触发
- 低风险(≥60): 3轮触发
```

## 三、AI Prompt构建

### 输入数据结构
```json
{
  "positions": [
    {
      "port": 9094,
      "trade_id": 12345,
      "pair": "BTC/USDT",
      "direction": "long",
      "entry_price": 95000.00,
      "current_price": 94800.00,
      "pnl_pct": -0.21,
      "leverage": 10,
      "atr_14": 150.0
    }
  ],
  "blacklist": [
    {"pair": "DOT/USDT", "level": "warn"}
  ]
}
```

### Prompt模板
```
当前持仓扫描（{count}个）:
=== 持仓 {pair} @ 端口{port} ===
  方向: {direction} | 杠杆: {leverage}x
  开仓价: {entry_price} | 当前价: {current_price}
  浮亏/盈: {pnl_pct}%
  TradeID: {trade_id}

黑名单状态: {blacklist}

请分析以上持仓，对每个持仓给出决策JSON。
评分规则:
- <40分: 高危 → full_close（立即全平）
- 40-59分: 中危 → half_close（半仓）
- 60-79分: 低风险 → observe（观察）
- ≥80分: 安全 → no_action

返回格式（JSON数组，每个持仓一行）:
{"trade_id":xxx,"pair":"xxx","port":xxx,"score":xx,"confidence":xx,"alert_type":"xxx","action":"xxx","reason":"xxx","feishu_alert":false}
如果全部无需操作: {"action":"no_action","reason":"所有持仓安全"}
```

## 六、轮次门控实现

```
每个(端口,交易对,TradeID)为独立key
连续出现在扫描中 → round+1
本轮未出现 → round重置为0
持仓消失后重新出现 → 从1重新计数

评分<40: 1轮即执行
评分40-59: 需2轮确认
评分≥60: 需3轮确认
```

## 七、评分系统（AI内部决策）

### 方向错误检测（5选3，每条20分，满分100）

| # | 信号 | 做多条件 | 做空条件 |
|---|------|---------|---------|
| 1 | RSI极端区 | RSI>70 + 浮亏>1% | RSI<30 + 浮亏>1% |
| 2 | RSI背离 | RSI<50 且下行 | RSI>50 且上行 |
| 3 | S/R破位 | 跌破支撑>1% | 突破阻力>1% |
| 4 | 缩量确认 | 连续3根量比<1.0 | 连续3根量比<1.0 |
| 5 | 铁律违反 | 不在支撑±1% | 不在压力±2% |

### 过早入场检测（4选2，每条25分，满分100）

| # | 信号 | 阈值 |
|---|------|------|
| 1 | 入场距S/R距离 | >3% |
| 2 | 入场量比 | <2.5x |
| 3 | R3过早判定 | true |
| 4 | 入场RSI | 非极端值 |
