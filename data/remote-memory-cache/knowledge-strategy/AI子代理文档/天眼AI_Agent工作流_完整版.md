### AI 决策规则（完整版）

#### 2.1 评分系统

**方向错误检测（5选3，每条20分，满分100）：**

| # | 信号 | 做多条件 | 做空条件 |
|---|------|---------|---------|
| 1 | RSI极端区 | RSI>70 + 浮亏>1% | RSI<30 + 浮亏>1% |
| 2 | RSI背离 | RSI<50 且下行 | RSI>50 且上行 |
| 3 | S/R破位 | 跌破支撑>1% | 突破阻力>1% |
| 4 | 缩量确认 | 连续3根量比<1.0 | 连续3根量比<1.0 |
| 5 | 铁律违反 | 不在支撑±1% | 不在压力±2% |

**过早入场检测（4选2，每条25分，满分100）：**

| # | 信号 | 阈值 |
|---|------|------|
| 1 | 入场距S/R距离 | >3%（中间地带） |
| 2 | 入场量比 | <2.5x |
| 3 | R3过早判定 | r3.is_premature == true |
| 4 | 入场RSI | 非极端值（40-70之间） |

#### 2.2 分级轮次门控

| 评分区间 | 风险等级 | 轮次要求 | 执行时机 |
|---------|---------|---------|---------|
| <40分 | 🔴 高危 | **1轮触发** | 立即执行 |
| 40-59分 | 🟠 中高危 | **2轮触发** | 第2轮确认后执行 |
| ≥60分 | 🟡 中风险 | **3轮触发** | 第3轮确认后执行 |

**轮次计数器逻辑：**
- 每个"端口-交易对"为独立key
- 同key连续出现在扫描中则+1轮
- 被标记但本轮未出现在扫描中 → 重置为0
- 持仓消失后重新出现 → 从1重新计数

#### 2.3 干预动作矩阵

**过早入场（方向正确，时机错误）：**

| 评分 | 持仓状态 | AI决策 |
|------|---------|--------|
| 60-79 | 方向对，强信号 | `observe` — 保持观察，等信号确认 |
| 40-59 | 方向对但弱 | `observe` + 监控补仓线 |
| <40 | 方向对但弱 | `half_close` + 设置L1补仓线 |

**方向错误：**

| 评分 | 持仓状态 | AI决策 |
|------|---------|--------|
| 50-79 | 方向错但强势 | `half_close` — 半仓，等反弹平 |
| <50 | 方向错且弱势 | `full_close` — **立即全平** |

**方向错误 + 多重违反（评分<40 + ≥4条违反）：**
- `full_close` — 立即全平
- 标记为「高危仓位」
- 14天内同一交易对入场权重+20

#### 2.4 ATR动态补仓参数

```
ATR(14) = 最近14根K线的 True Range 平均值

L1补仓触发：亏损额 ≥ 0.5 × ATR(14)
L2补仓触发：亏损额 ≥ 1.0 × ATR(14)
补仓量：当前仓量 × 0.5
最大补仓次数：2次
补仓冷却：≥30分钟
补仓后止损：均摊价的 -5%
```

**各交易对参考值：**

| 交易对 | ATR(14) | L1触发 | L2触发 |
|--------|---------|--------|--------|
| BTC/USDT | $150 | -$75 | -$150 |
| ETH/USDT | $15 | -$7.5 | -$15 |
| SOL/USDT | $0.8 | -$0.4 | -$0.8 |
| DOGE/USDT | $0.003 | -$0.0015 | -$0.003 |
| DOT/USDT | $0.05 | -$0.025 | -$0.05 |

## 六、DOT黑名单工作流

```
持仓平仓时触发：
    │
    ▼
读取 realized_pnl
    │
    ├── pnl < 0 → loss_count + 1
    ├── pnl >= 0 → win_count + 1
    │
    ▼
计算胜率 = win_count / total_trades
    │
    ├── loss_count >= 7 或 胜率 <= 25% → 🔴 黑名单（禁止新开仓）
    ├── loss_count >= 5 或 胜率 <= 30% → 🟠 警告名单（建议观望）
    └── loss_count >= 3              → 🟡 观察名单（加严入场）
    │
    ▼
更新 dot_blacklist 表 + 通知飞书
```

## 七、准确率追踪工作流

```
预警触发时：
    │
    ▼
记录 alert_records：
  trigger_ts = now
  score = AI评分
  action_taken = 执行动作
  price_at_trigger = 当前价格
    │
    ▼
5分钟后（定时任务）：
    │
    ▼
更新 alert_records：
  price_after_5min = 5分钟后的价格
  reversed_5min = true（如果向有利方向移动>0.5×ATR）
    │
    ▼
平仓时：
    │
    ▼
更新 alert_records：
  price_at_close = 平仓价格
  realized_pnl = 实际盈亏
  correct = 判定（综合5min/10min反转 + 最终盈亏）
    │
    ▼
每日汇总 accuracy_daily：
  - 按交易对准确率
  - 按预警类型准确率
  - 按执行动作准确率
```
