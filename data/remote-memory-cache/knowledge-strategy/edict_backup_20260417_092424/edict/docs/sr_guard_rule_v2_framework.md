# S/R方向守卫 · 规则框架 V2

## 规则目标
持仓方向必须与入场时的S/R位置一致，方向错误立即平仓。

## 核心参数
- SR_TOLERANCE_PCT = 3.0  （S/R区间 ±3%）
- MAX_POSITION_AGE_SEC = 24小时（L1年龄过滤，超过24小时不告警。太子批复：原1h过于严格，改为24h。2026-03-31）
- COOLDOWN_SEC = 30分钟（同一违规，30分钟冷却）

---

## 规则一：S/R方向错误（非支撑位下空 / 压力位上多 → 平仓）

### LONG 规则
- 条件：当前价在**压力位±3%**内（距压力 < 3%）
- 动作：平仓

### SHORT 规则
- 条件：当前价在**支撑位±3%**内（距支撑 < 3%）
- 动作：平仓

### 判断逻辑
```
LONG: d2res = (压力 - 当前价) / 当前价 × 100
     if d2res < 3.0% → 平仓

SHORT: d2s = (当前价 - 支撑) / 当前价 × 100
     if d2s < 3.0% → 平仓
```

---

## 规则二（补救条款）：S/R方向错误 + 负收益 → 平仓

### LONG 补救规则
- 条件1：当前价在**支撑±3%**内（价格已跌到支撑附近，但持仓方向是做多）
- 条件2：当前亏损（当前价 < 入场价）
- 动作：平仓

### SHORT 补救规则
- 条件1：当前价在**压力±3%**内（价格已涨到压力附近，但持仓方向是做空）
- 条件2：当前亏损（当前价 > 入场价）
- 动作：平仓

### 判断逻辑
```
LONG: 
  - d2sup = (当前价 - 支撑) / 当前价 × 100
  - d2sup < 3.0% → 在支撑区间
  - 当前价 < 入场价 → 亏损中
  → 两个都满足 → 平仓

SHORT:
  - d2res = (压力 - 当前价) / 当前价 × 100
  - d2res < 3.0% → 在压力区间
  - 当前价 > 入场价 → 亏损中
  → 两个都满足 → 平仓
```

---

## L2 入场位置过滤（规则一加强）

入场时方向正确（在正确S/R位附近开仓），即使当前偏离也不告警：
- LONG：入场价在**支撑±3%**内 → 入场正确，跳过
- SHORT：入场价在**压力±3%**内 → 入场正确，跳过

## 状态机

```
扫描持仓
  ├── 获取入场时间（open_date）
  ├── 获取当前S/R（Gate.io K线）
  └── 逐条判断：
        ├── L1: 年龄 > 1小时 → 跳过
        ├── L2: 入场位置正确 → 跳过
        ├── 规则1触发？ → 写提案（平仓）
        ├── 规则2触发？ → 写提案（补救平仓）
        └── 无触发 → 跳过
```

---

## 优先级

规则二（补救条款）**优先级高于**规则一：
- 规则一是"当前在错误S/R位"→ 平仓
- 规则二是"在错误S/R位 + 仍亏损"→ 平仓（补救）

两个规则都触发时，只发一次提案。
