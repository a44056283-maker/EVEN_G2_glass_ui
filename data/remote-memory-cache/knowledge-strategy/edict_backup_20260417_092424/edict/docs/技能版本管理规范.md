## 二、技能更新标准流程

### 2.1 更新前准备

```bash
# 1. 确认当前版本
cat ~/.openclaw/skills/<skill-name>/_meta.json

# 2. 查看 gitops 快照历史（如有）
ls ~/.openclaw/gitops/<skill-name>/snapshots/ 2>/dev/null

# 3. 记录更新原因
echo "$(date -I) 更新原因: xxx" >> ~/.openclaw/gitops/<skill-name>/changelog.md
```

### 2.2 标准更新步骤

```
┌─────────────────────────────────────────────┐
│  步骤1: 快照当前状态（deploy前必做）           │
└─────────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────┐
│  步骤2: 下载/替换新版本 SKILL.md              │
└─────────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────┐
│  步骤3: 更新 _meta.json version 字段          │
└─────────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────┐
│  步骤4: 运行 arc-trust-verifier 审计          │
└─────────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────┐
│  步骤5: 测试验证（功能正常）                   │
└─────────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────┐
│  步骤6: 提交快照到 ~/.openclaw/gitops/        │
└─────────────────────────────────────────────┘
```

### 2.3 arc-skill-gitops 框架命令参考

> 框架脚本位于 `~/.openclaw/skills/arc-skill-gitops/scripts/`（注：当前该目录为空，框架尚未完全部署）

```bash
# 初始化技能跟踪
python3 ~/.openclaw/skills/arc-skill-gitops/scripts/gitops.py init --skill ~/.openclaw/skills/<name>/

# 拍摄快照（更新前必做）
python3 ~/.openclaw/skills/arc-skill-gitops/scripts/gitops.py snapshot --skill ~/.openclaw/skills/<name>/ --tag "pre-v1.x.x"

# 部署更新（自动快照）
python3 ~/.openclaw/skills/arc-skill-gitops/scripts/gitops.py deploy --skill ~/.openclaw/skills/<name>/ --tag "v1.x.x"

# 查看快照历史
python3 ~/.openclaw/skills/arc-skill-gitops/scripts/gitops.py history --skill ~/.openclaw/skills/<name>/

# 状态检查
python3 ~/.openclaw/skills/arc-skill-gitops/scripts/gitops.py status
```

## 三、快照/回滚操作指南

### 3.1 手动快照流程

```bash
SKILL_NAME="arc-skill-gitops"
SNAPSHOT_DIR="$HOME/.openclaw/gitops/$SKILL_NAME/snapshots"
mkdir -p "$SNAPSHOT_DIR"

TAG="pre-$(date +%Y%m%d-%H%M%S)"
DEST="$SNAPSHOT_DIR/$TAG"
mkdir -p "$DEST"

# 复制 SKILL.md 和所有文件
cp -r ~/.openclaw/skills/$SKILL_NAME/* "$DEST/"

# 记录快照元数据
cat > "$DEST/snapshot.meta" << EOF
{
  "tag": "$TAG",
  "created_at": "$(date -Iseconds)",
  "skill": "$SKILL_NAME",
  "version": "$(cat ~/.openclaw/skills/$SKILL_NAME/_meta.json | python3 -c 'import sys,json; print(json.load(sys.stdin).get("version","unknown"))')"
}
EOF

echo "✅ 快照已保存: $TAG"
```

### 3.2 回滚操作

```bash
SKILL_NAME="arc-skill-gitops"
SNAPSHOT_TAG="pre-20260327-120000"  # 替换为实际tag

# 确认快照存在
ls "$HOME/.openclaw/gitops/$SKILL_NAME/snapshots/$SNAPSHOT_TAG/" || exit 1

# 执行回滚（覆盖当前版本）
cp -r "$HOME/.openclaw/gitops/$SKILL_NAME/snapshots/$SNAPSHOT_TAG/"* \
  ~/.openclaw/skills/$SKILL_NAME/

# 恢复版本号（如需要）
echo "✅ 回滚完成: $SNAPSHOT_TAG"
```

### 3.3 紧急回滚 Checklist

- [ ] 确认问题症状和发生时间
- [ ] 确定最近的可信快照
- [ ] 执行回滚命令
- [ ] 验证核心功能正常
- [ ] 如有必要，通知尚书省和受影响部门
- [ ] 记录事故报告

## 四、工部技能管理 Checklist

### 4.1 每次更新前（必做）

- [ ] 拍摄当前状态快照
- [ ] 阅读新版本 SKILL.md 变更内容
- [ ] 确认无破坏性变更
- [ ] 确认依赖满足（python3, git 等）
- [ ] 通知尚书省即将更新

### 4.2 更新后验证（必做）

- [ ] 运行 `arc-trust-verifier` 审计通过
- [ ] 核心功能测试通过
- [ ] 无新增危险权限
- [ ] 更新 `_meta.json` 版本号
- [ ] 记录到 changelog
- [ ] 上报尚书省完成情况

### 4.3 定期检查（每月）

- [ ] 检查所有技能版本是否有更新
- [ ] 审计所有技能安全（arc-trust-verifier）
- [ ] 清理过期快照（保留最近6个）
- [ ] 更新本规范文档

### 4.4 禁止事项

- ❌ 禁止在未经快照的情况下直接覆盖技能文件
- ❌ 禁止安装来源不明的技能
- ❌ 禁止跳过 trust-verifier 审计直接部署
- ❌ 禁止修改尚书省分配的技能目录结构
