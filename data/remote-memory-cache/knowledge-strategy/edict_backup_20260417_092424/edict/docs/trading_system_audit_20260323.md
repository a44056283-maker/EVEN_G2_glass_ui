## 二、入场规则（L1-L5 规则引擎）

> 核心原则：**资金流方向 → 开仓方向；量比 → 杠杆倍数；S/R 位置 → 入场确认**

### L1 噪音过滤
- 量比 = 当前成交量 / 过去96根K线均量（垃圾时间基线）
- 量比 < 1.5x → 判定为噪音，**跳过**
- 量比 >= 1.5x → 有效信号，继续 L2

### L2 资金流方向
- Gate.io: 读取 `taker_long_short_ratio`（多空主动买入量比）
- net_flow_ratio = long_taker / short_taker
- net_flow_ratio > 1.0 → 净多头 → 允许做多 LONG
- net_flow_ratio < 1.0 → 净空头 → 允许做空 SHORT
- OI 变化率 > 1.5% → 确认趋势方向

### L3 信号捕捉（K线形态）
- 触底次数（支撑位被测试次数）≥ 1次 → 做多候选
- 触顶次数（压力位被测试次数）≥ 1次 → 做空候选
- RSI 超卖 < 25 + 布林下轨 → 做多超卖信号（绕过评分过滤）
- RSI 超买 > 75 + 布林上轨 → 做空超买信号

### L4 支撑/压力位（S/R Engine）
- 多时间框架（15m/30m/1h/4h）K线 → 计算支撑位和压力位
- 做多：价格必须在支撑位附近（±2%内）+ 距支撑 < 10%
- 做空：价格必须在压力位附近（±2%内）+ 距压力 < 10%
- 支撑/压力位距离 > 10% → **严格禁止入场**
- 无 S/R 数据 → **严格禁止入场**

### L5 仓位管理（动态杠杆）
| 信号强度 | 触发条件 | 杠杆 |
|---------|---------|------|
| 强信号 (strong) | 巨量 > 10x + 净流入确认 | 10x |
| 中信号 (medium) | 强量 > 6x 或 S/R 强确认 | 8x |
| 弱信号 (weak) | 有效量 > 3x | 5x |

- 仓位 = 余额 × 5% × 量比倍数 × 对锁系数（对锁 = 0.5，无 = 1.0）
- 最小保证金: $10
- 最大敞口: 余额的 100%

## 三、出场规则（L0-L5）

### L0 28定律（保命底线）
| 杠杆 | 允许亏损（本金比例）| 触发强平 |
|------|-------------------|---------|
| 10x | 80% → 实际价格偏移 8% | 强制平仓 |
| 8x  | 60% → 实际价格偏移 7.5% | 强制平仓 |
| 5x  | 40% → 实际价格偏移 8% | 强制平仓 |

### L1 动态止盈（分批锁利）
**按杠杆分层，每档触发只卖一次：**

10x（激进）:
- 盈利 ≥ 8% → 卖出 20%
- 盈利 ≥ 15% → 卖出 30%（累计50%）
- 盈利 ≥ 25% → 卖出 50%（累计75%，留25%跑）

8x:
- 盈利 ≥ 10% → 卖出 15%
- 盈利 ≥ 18% → 卖出 25%（累计40%）
- 盈利 ≥ 30% → 卖出 40%（累计64%）

5x:
- 盈利 ≥ 15% → 卖出 10%
- 盈利 ≥ 25% → 卖出 20%
- 盈利 ≥ 40% → 卖出 30%
- 盈利 ≥ 60% → 卖出 40%

### L2 时线止盈（支撑/压力位检测）
- 启用时线: 30m / 1h / 4h（可配置）
- 自动模式: ≥5x 杠杆 → 中短线（30分+1小时）；< 5x → 中长线（4小时+1日）
- 做多: 价格触及压力位 + 放量确认（量比 > 1.5x）→ 止盈
- 做空: 价格触及支撑位 + 放量确认 → 止盈
- 各时线止盈比例可独立配置

### L3 Trigger 九层审计（入场前最后关卡）
1. 滑点审计
2. 流动性审计
3. 持仓超限审计
4. 周末风险审计
5. 合约大小审计
6. 流动性深度审计
7. 极端行情审计
8. 资金费率审计
9. 综合风险评级

### L4 平仓冷却
- 同币种止损/止盈后 **5分钟内禁止重新入场**（防止反复开仓）
