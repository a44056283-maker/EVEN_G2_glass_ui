## 一、缠论哲学

### 1.1 缠论是什么

缠论是**无需预测**的交易方法。

它不预测顶底，不猜涨跌。它只是**分类**：

> "市场的所有走势，只有上涨、下跌、盘整三种。"
> — 缠中说禅

### 1.2 缠论的核心思想

| 原则 | 含义 |
|------|------|
| **不预测，只分类** | 任何走势只有三种可能，提前想好对策 |
| **走势必完美** | 任何走势一定会走完，涨完了必跌，跌完了必涨 |
| **利用市场弱点** | 主力无法改变走势必完美的规律 |

### 1.3 缠论 vs 其他理论

| 理论 | 核心 | 缠论优势 |
|------|------|---------|
| 波浪理论 | 5浪推动+3浪调整 | 缠论更精确，笔/线段有严格定义 |
| 道氏理论 | 趋势定义模糊 | 缠论笔/线段有数学定义 |
| 支撑阻力 | 主观画线 | 缠论中枢有精确重叠区间算法 |

## 二、分型 — 缠论的DNA

### 2.1 分型定义

分型是**3根连续K线**构成的结构，是缠论的最小构件。

```
     K线1        K线2        K线3
      │           │           │
      ▼           ▼           ▼
    ┌───┐       ┌───┐
────┘   └───────┘   └───────  顶分型（中间最高）

    ┌───┐       ┌───┐
────┘   └───────┘   └───────  底分型（中间最低）
```

### 2.2 顶分型

**结构**：三根K线，中间K线的高点最高

```
条件：H2 > H1 且 H2 > H3
（H1=第1根高点，H2=第2根高点，H3=第3根高点）
```

**含义**：多方力量在第2根K线达到顶峰后衰竭

### 2.3 底分型

**结构**：三根K线，中间K线的低点最低

```
条件：L2 < L1 且 L2 < L3
（L1=第1根低点，L2=第2根低点，L3=第3根低点）
```

**含义**：空方力量在第2根K线达到顶峰后衰竭

### 2.4 包含关系处理

**定义**：相邻两根K线，一根被另一根完全包含

```
包含关系示例：
    ┌─────┐
────┘     └────  ← 这两根K线有包含关系
```

**处理规则**：

| 趋势方向 | 处理方法 | 结果 |
|---------|---------|------|
| 上升趋势（底分型附近） | 取两K线的**高高** | 合并后的K线高点取max，低点也取max |
| 下降趋势（顶分型附近） | 取两K线的**低低** | 合并后的K线高点取min，低点也取min |

**处理步骤**：
1. 找到有包含关系的相邻K线
2. 判断当前趋势方向（以上一个分型为参考）
3. 按规则合并K线
4. 重复处理，直到没有任何包含关系

### 2.5 Python实现

```python
class FractalAnalyzer:
    """分型识别器"""
    
    def __init__(self, klines):
        """
        klines: list of (timestamp, open, high, low, close)
        """
        self.klines = klines
    
    def process_containing(self):
        """处理包含关系，返回处理后的K线"""
        processed = []
        i = 0
        
        while i < len(self.klines):
            if len(processed) == 0:
                processed.append(self.klines[i])
                i += 1
                continue
            
            prev = processed[-1]  # 前一根处理后的K线
            curr = self.klines[i]  # 当前原始K线
            
            # 判断是否有包含关系
            if self._has_containing(prev, curr):
                # 合并
                merged = self._merge(prev, curr, processed)
                processed[-1] = merged
            else:
                processed.append(curr)
            
            i += 1
        
        return processed
    
    def _has_containing(self, k1, k2):
        """判断两根K线是否有包含关系"""
        h1, l1 = k1[2], k1[3]  # high, low
        h2, l2 = k2[2], k2[3]
        
        # 一根完全高于另一根
        return (h1 >= h2 and l1 <= l2) or (h1 <= h2 and l1 >= l2)
    
    def _merge(self, k1, k2, processed):
        """合并有包含关系的K线"""
        h1, l1 = k1[2], k1[3]
        h2, l2 = k2[2], k2[3]
        
        # 判断趋势：看processed中倒数第二根的方向
        if len(processed) >= 2:
            prev_prev = processed[-2]
            hpp, lpp = prev_prev[2], prev_prev[3]
            # 如果前K线在上涨，取高高
            if hpp > lpp:
                new_h = max(h1, h2)
                new_l = max(l1, l2)
            else:
                new_h = min(h1, h2)
                new_l = min(l1, l2)
        else:
            # 第一根，直接取高高
            new_h = max(h1, h2)
            new_l = max(l1, l2)
        
        return (k1[0], k1[1], new_h, new_l, k2[4])
    
    def find_fractals(self, klines):
        """找所有分型（处理包含关系后）"""
        processed = self.process_containing()
        fractals = []
        
        for i in range(1, len(processed) - 1):
            prev = processed[i-1]
            curr = processed[i]
            next_ = processed[i+1]
            
            h1, l1 = prev[2], prev[3]
            h2, l2 = curr[2], curr[3]
            h3, l3 = next_[2], next_[3]
            
            # 顶分型
            if h2 > h1 and h2 > h3:
                fractals.append({
                    'type': 'top',
                    'index': i,
                    'high': h2,
                    'timestamp': curr[0]
                })
            
            # 底分型
            elif l2 < l1 and l2 < l3:
                fractals.append({
                    'type': 'bottom',
                    'index': i,
                    'low': l2,
                    'timestamp': curr[0]
                })
        
        return fractals
```

## 三、笔 — 短期趋势的载体

### 3.1 笔的定义

**笔 = 顶分型 + 至少1根独立K线 + 底分型**（或反向）

```
上升笔结构：
底分型 → 至少1根K线 → 顶分型
   ↑                      ↑
 起点                   终点

下降笔结构：
顶分型 → 至少1根K线 → 底分型
   ↑                      ↑
 起点                   终点
```

### 3.2 笔的规则

| 规则 | 说明 |
|------|------|
| 顶分型后必然是底分型 | 笔的方向必然交替 |
| 笔必须独立 | 顶底分型之间至少1根不属于任何分型的K线 |
| 笔的幅度 | 顶底之间应有明显幅度（无固定数值要求） |
| 笔破坏 | 原笔方向被反向笔打破 = 笔终结 |

### 3.3 笔的Python识别

```python
class StrokeAnalyzer:
    """笔识别器"""
    
    def __init__(self, klines):
        self.fractal_analyzer = FractalAnalyzer(klines)
        self.klines = klines
    
    def find_strokes(self):
        """识别所有笔"""
        fractals = self.fractal_analyzer.find_fractals(self.klines)
        if len(fractals) < 2:
            return []
        
        strokes = []
        i = 0
        
        while i < len(fractals) - 1:
            f1 = fractals[i]
            f2 = fractals[i + 1]
            
            # 检查是否有足够的独立K线
            start_idx = f1['index']
            end_idx = f2['index']
            
            # 至少1根独立K线
            if abs(end_idx - start_idx) >= 2:
                if f1['type'] == 'bottom' and f2['type'] == 'top':
                    # 上升笔
                    strokes.append({
                        'type': 'up',
                        'start': start_idx,
                        'end': end_idx,
                        'start_price': f1.get('low', 0),
                        'end_price': f2.get('high', 0),
                        'amplitude': f2.get('high', 0) - f1.get('low', 0)
                    })
                elif f1['type'] == 'top' and f2['type'] == 'bottom':
                    # 下降笔
                    strokes.append({
                        'type': 'down',
                        'start': start_idx,
                        'end': end_idx,
                        'start_price': f1.get('high', 0),
                        'end_price': f2.get('low', 0),
                        'amplitude': f1.get('high', 0) - f2.get('low', 0)
                    })
            
            i += 1
        
        return strokes
```

## 四、线段 — 中期趋势的结构

### 4.1 线段定义

**线段 = 连续3笔以上，同方向**

```
上升线段：
笔1(上升) + 笔2(下降) + 笔3(上升) = 上升线段
笔1被笔2破坏（顶底互换）= 线段终结
```

### 4.2 线段被破坏

线段的破坏必须满足：
1. **被反向笔破坏**（不充分）
2. **被反向线段破坏**（充分）

```
线段破坏示例：

上升线段：笔1↑ + 笔2↓ + 笔3↑ + 笔4↓ + 笔5↑
                              ↓
                         笔4↑(反向笔) → 笔被破坏，但线段未破坏
                               ↓
                         笔6↓(反向线段) → 线段被完全破坏
```

### 4.3 线段的实战意义

| 场景 | 信号 |
|------|------|
| 上升线段不断创新高 | 趋势强势，继续持有 |
| 上升线段内部笔背驰 | 可能转折预警 |
| 线段被反向线段破坏 | 趋势反转，确认 |

## 五、中枢 — 缠论的核心

### 5.1 中枢定义

**中枢 = 至少3段连续笔的重叠区间**

```
       笔1 ──┐
             │
  笔3 ──────┼──→ 重叠区间 = [max(笔1低, 笔3低), min(笔2高, 笔4高)]
             │
       笔2 ──┘

重叠区间 = MAX(笔1.low, 笔3.low) ~ MIN(笔2.high, 笔4.high)
```

### 5.2 中枢的Python实现

```python
class CenterAnalyzer:
    """中枢识别器"""
    
    def find_center_zone(self, strokes):
        """
        识别中枢
        中枢 = 连续3笔的重叠区间
        """
        if len(strokes) < 3:
            return []
        
        centers = []
        i = 0
        
        while i <= len(strokes) - 3:
            s1, s2, s3 = strokes[i], strokes[i+1], strokes[i+2]
            
            # 检查三笔是否有重叠
            highs = [s1['end_price'], s2['end_price']]
            lows = [s1['start_price'], s2['start_price']]
            
            # 三笔方向相同才构成中枢
            if s1['type'] == s2['type'] == s3['type']:
                # 取这三笔的极值
                high_max = max(s1.get('end_price', 0), s1.get('start_price', 0),
                              s2.get('end_price', 0), s2.get('start_price', 0),
                              s3.get('end_price', 0), s3.get('start_price', 0))
                low_min = min(s1.get('end_price', 0), s1.get('start_price', 0),
                             s2.get('end_price', 0), s2.get('start_price', 0),
                             s3.get('end_price', 0), s3.get('start_price', 0))
                
                # 有重叠 = 高点高于前低点
                if high_max > low_min:
                    centers.append({
                        'start_stroke': i,
                        'end_stroke': i + 2,
                        'high': high_max,
                        'low': low_min,
                        'type': s1['type']
                    })
            
            i += 1
        
        return centers
```

### 5.3 中枢的分类

| 类型 | 特征 | 实战意义 |
|------|------|---------|
| **上升中枢** | 高低点逐步抬高 | 中枢上移，看涨 |
| **下降中枢** | 高低点逐步降低 | 中枢下移，看跌 |
| **扩张中枢** | 高点抬高+低点降低同时出现 | 震荡加剧，可能反转 |
| **中枢延伸** | 9段以上延伸 | 趋势延续但力度减弱 |

### 5.4 中枢实战应用

#### 5.4.1 区间交易

在中枢内：
- **高抛**：价格触及中枢上沿
- **低吸**：价格触及中枢下沿

```
           ┌─── 上沿 ───┐
  价格 ────┤             ├── 中枢区间
           └─── 下沿 ───┘
           ↑低吸         ↑高抛
```

#### 5.4.2 突破确认

| 情况 | 信号 | 操作 |
|------|------|------|
| 突破上沿后回踩不破 | 有效突破 | 买入 |
| 跌破下沿后反弹不破 | 有效跌破 | 卖出 |
| 突破后返回中枢内 | 假突破 | 观望 |

#### 5.4.3 中枢背驰

**背驰 = 离开中枢的力度 < 进入中枢的力度**

```
背驰示例：
笔1(进入中枢): 幅度大，力度强
笔2(离开中枢): 幅度小，力度弱 → 背驰 = 可能反转
```

## 六、背驰 — 判断转折的核心

### 6.1 背驰定义

**背驰 = 离开中枢的笔比进入中枢的笔力度弱**

判断方法：
1. **MACD辅助**：离开中枢的MACD面积 < 进入中枢的MACD面积
2. **幅度辅助**：离开中枢的K线幅度 < 进入中枢的K线幅度

### 6.2 背驰类型

| 类型 | 条件 | 信号 |
|------|------|------|
| 上升中继背驰 | 离开中枢的上升笔幅度 < 进入的幅度 | 上涨衰竭，可能反转 |
| 上升转折背驰 | 离开中枢的上升笔幅度 < 进入的幅度 | 确认下跌开始 |
| 下降中继背驰 | 离开中枢的下降笔幅度 < 进入的幅度 | 下跌衰竭，可能反弹 |
| 下降转折背驰 | 离开中枢的下降笔幅度 < 进入的幅度 | 确认上涨开始 |

### 6.3 Python背驰检测

```python
def detect_divergence(self, strokes, center):
    """检测背驰"""
    if len(strokes) < center['end_stroke'] + 2:
        return None
    
    # 进入中枢的笔
    entry_stroke = strokes[center['start_stroke']]
    # 离开中枢的笔
    exit_stroke = strokes[center['end_stroke'] + 1]
    
    # 计算幅度
    entry_amp = abs(entry_stroke['amplitude'])
    exit_amp = abs(exit_stroke['amplitude'])
    
    # 计算MACD面积（需要附加MACD数据）
    
    if center['type'] == 'up':
        # 上升趋势中离开的笔幅度变小的 = 背驰
        if exit_stroke['type'] == 'up' and exit_amp < entry_amp:
            return {'type': 'bearish_divergence', 'strength': entry_amp / exit_amp}
    else:
        # 下降趋势中离开的笔幅度变小的 = 背驰
        if exit_stroke['type'] == 'down' and exit_amp < entry_amp:
            return {'type': 'bullish_divergence', 'strength': entry_amp / exit_amp}
    
    return None
```

## 七、买卖点 — 量化操作

### 7.1 一买/一卖（第一类）

**一买**：下降背驰后的最低点

**一卖**：上升背驰后的最高点

```
一买：
下跌笔背驰 → 最低点 = 第一类买点
                    ↑
           一买确认后 → 二买出现
```

### 7.2 二买/二卖（第二类）

**二买**：一买后反弹，再次回踩不破新低

**二卖**：一卖后下跌，再次反弹不创新高

```
二买：
一买之后 ──→ 反弹 ──→ 回踩 ──→ 不破一买低点 = 二买
                                    ↑
                              更安全的买点
```

### 7.3 三买/三卖（第三类）

**三买**：向上突破中枢后，回踩不进入中枢

**三卖**：向下跌破中枢后，反弹不进入中枢

```
三买：
突破中枢上沿 → 回踩 → 不回到中枢内 = 三买 ← 爆发力最强
```

### 7.4 买卖点对比

| 买点 | 安全性 | 成本 | 爆发力 | 操作难度 |
|------|-------|------|-------|---------|
| 一买 | 低 | 最低 | 中 | 极高（需判断背驰） |
| 二买 | 中 | 中 | 高 | 中（需等待确认） |
| 三买 | 高 | 高 | 最高 | 低（确认突破后买） |

## 八、综合实战流程

### 8.1 选币流程

```
1. 看周线/日线：找长期趋势方向
   ↓
2. 找周线/日线上的分型/笔/线段
   ↓
3. 画周线/日线的中枢
   ↓
4. 判断是否在买点上
   ↓
5. 切换到1H/4H找精确入场点
   ↓
6. 用分型确认
   ↓
7. 入场
```

### 8.2 止损设置

| 入场点 | 止损位置 |
|-------|---------|
| 一买 | 低于一买最低点 |
| 二买 | 低于一买最低点 或 中枢下沿 |
| 三买 | 中枢下沿 |

### 8.3 持仓管理

```
入场 ──→ 涨势良好 ──→ 持有
       │
       ├──→ 出现背驰 ──→ 减仓50%
       │
       └──→ 跌破上升笔低点 ──→ 全部平仓
```

## 九、常见错误

| 错误 | 后果 | 纠正 |
|------|------|------|
| 在下降趋势中抄底买一买 | 被套 | 只在周线/日线上涨趋势中操作 |
| 不处理包含关系 | 分型误判 | 必须先处理包含关系再找分型 |
| 把笔和线段搞混 | 方向判断错误 | 笔=3根K线分型，线段=3笔 |
| 中枢区间画错 | 买卖点全错 | 用重叠区间算法，不是主观画线 |
| 背驰判断太主观 | 假背驰被骗 | 用MACD面积+幅度双重确认 |
