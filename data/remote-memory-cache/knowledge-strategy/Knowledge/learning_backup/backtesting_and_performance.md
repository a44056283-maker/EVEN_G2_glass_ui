## 一、回测陷阱（必须避免）

### 1.1 过拟合 (Overfitting)
**定义**: 策略过度适应历史数据的噪音，而非捕捉真实规律

**识别信号**:
- 回测夏普比率极高(>3)
- 参数组合过于精确（如EMA_42而非EMA_40）
- 样本外测试表现大幅下降

**规避方法**:
- 使用简单参数
- 样本内/样本外分离验证
- walk-forward分析

### 1.2 前视偏差 (Look-ahead Bias)
**定义**: 使用了回测时刻点不该知道的信息

**常见场景**:
- 使用收盘价作为入场信号（收盘时才能确定）
- 未来函数
- 偷价格（peek）

**规避方法**:
- 使用上一根K线收盘价作为信号
- 信号确认后再下单

### 1.3 生存者偏差 (Survivorship Bias)
**定义**: 只用现存标的回测，排除已退市/破产标的

**影响**: 虚高收益10-30%

**规避方法**:
- 使用包含已退市标的的历史数据库
- 定期更新股票池

### 1.4 非流动性偏差
**定义**: 回测假设能以收盘价成交，但实际滑点大

**规避方法**:
- 模拟限价单成交
- 加入交易成本（滑点0.1-0.5%）

## 二、核心绩效指标

### 2.1 夏普比率 (Sharpe Ratio)
**公式**: (平均收益 - 无风险利率) / 收益标准差

**评估标准**:
| 夏普比率 | 评价 |
|---------|------|
| < 0.5 | 差 |
| 0.5-1.0 | 一般 |
| 1.0-2.0 | 良好 |
| 2.0-3.0 | 优秀 |
| > 3.0 | 疑似过拟合 |

**简化版**: `夏普 = 年化收益 / 最大回撤`

### 2.2 索提诺比率 (Sortino Ratio)
**公式**: (平均收益 - 目标收益) / 下行收益标准差

**优势**: 只考虑下行波动，更准确反映风险

### 2.3 最大回撤 (Max Drawdown)
**定义**: 从峰值到谷值的最大跌幅

**测量方法**:
```
最大回撤 = (谷值 - 峰值) / 峰值 × 100%
```

**风险**: 
- 20%回撤需要25%收益回本
- 50%回撤需要100%收益回本
- 80%回撤需要400%收益回本

**回本难度公式**: `回本所需涨幅 = 1 / (1 - 回撤比例) - 1`

### 2.4 胜率与盈亏比
**期望值公式**: `E = 胜率 × 平均盈利 - 负率 × 平均亏损`

**最优平衡**:
- 高胜率(>60%) + 低盈亏比(>1.0) = 可行
- 低胜率(<40%) + 高盈亏比(>2.0) = 可行
- 胜率50% × 盈亏比1.0 = 原地踏步

## 三、户部绩效分析框架

### 3.1 核心数据提取
```sql
-- 胜率统计
SELECT 
  COUNT(*) as total_trades,
  SUM(CASE WHEN realized_profit > 0 THEN 1 ELSE 0 END) as wins,
  SUM(CASE WHEN realized_profit < 0 THEN 1 ELSE 0 END) as losses,
  AVG(CASE WHEN realized_profit > 0 THEN realized_profit END) as avg_win,
  AVG(CASE WHEN realized_profit < 0 THEN realized_profit END) as avg_loss,
  SUM(realized_profit) as total_pnl
FROM trades
WHERE close_date >= '2026-04-01';
```

### 3.2 绩效归因
**分析维度**:
1. 时间维度: 持仓时长 vs 收益
2. 方向维度: LONG vs SHORT 胜率对比
3. 币种维度: 各币种贡献度
4. 时段维度: 不同时间段表现

## 五、学习总结

**核心收获**:
1. 回测必须避免3大陷阱：过拟合、前视偏差、生存者偏差
2. 夏普比率>2才具备实盘价值
3. 户部策略盈亏比优秀(5:1)，但胜率(38.5%)偏低
4. 回撤控制是关键：20%回撤需25%回本

**实践建议**:
- 户部应优化入场信号，提升胜率
- L5动态止盈参数过高（04-10分析），应降低触发条件
- 监控每日夏普比率，保持在1.0以上
