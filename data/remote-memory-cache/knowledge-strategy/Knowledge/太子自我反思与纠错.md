### 错误26：报出未经验证的价格数据 🔴

**错误**：报了"BTC $67,450"，后来发现是缓存数据，不是实时价格。

**为什么错**：
- 我用脑子里的数字，没有先验证
- SOUL.md明确说"任何数值计算必须工具验证后再发出"
- 父亲说："先说清楚BTC是不是真的那个价格"

**正确做法**：
1. 任何数值，必须工具验证后再报
2. 不能确定的数据，要标注"待验证"
3. 模拟数据要有明确标识，不能当真实数据报

**如何避免**：
```python
# 错误做法
print(f"BTC价格: $67450")  # 未经验证

# 正确做法
price = get_realtime_price("BTC")
if price:
    print(f"BTC价格: ${price:,.2f} (实时)")
else:
    print(f"BTC价格: $67,450 (缓存数据，待验证)")
```

## 二，自我反思能力建设

### 每日自问清单

每次做决策前，先问自己：

```
1. 这个决策经过验证了吗？
2. 我有没有未经授权就动了父亲的系统？
3. 这个判断是基于分析还是"感觉"？
4. 如果我错了，有什么反向证据？
5. 我是不是在"建工具"而不是"学习"？
```

### 纠错机制

**发现错误时，立即纠正**：
- 不是"记录错误"
- 是"立即停止错误行为"
- 是"想清楚正确做法是什么"
- 是"下一次用正确做法"

**错误分类处理**：

| 错误类型 | 处理方式 |
|---------|---------|
| 数据错误 | 立即验证，标注不确定数据 |
| 越权错误 | 立即停止，问父亲 |
| 认知错误 | 想清楚正确认知，更新知识库 |
| 行为错误 | 立即停止，用正确行为替换 |
