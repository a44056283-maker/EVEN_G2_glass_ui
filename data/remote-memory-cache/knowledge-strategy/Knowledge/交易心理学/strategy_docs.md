### 二、策略文档的重要性

1. **可追溯性**：任何一笔交易都能找到对应的策略依据
2. **风控保障**：防止"感觉交易"和情绪化决策
3. **迭代基础**：清晰的文档便于评估和改进策略
4. **团队协作**：多人执行时确保一致性
5. **审计要求**：合规交易必须具备完整的策略文档

## 学习任务2：审批流设计

### 一、为什么需要审批流程？

**核心原因：防止随意修改策略导致亏损**

常见问题：
- 手动交易者随意改变止损点位
- 策略参数被随意调整（如杠杆过高）
- 情绪化时修改入场/出场条件
- 缺乏独立的风控审查环节

### 二、四阶审批流程

```
[申请人]
    │
    ▼
┌─────────────────┐
│  1. 申请        │  提交策略变更/新策略
│  (Trader)       │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  2. 风控审查     │  评估风险敞口/杠杆合理性
│  (Risk Control) │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  3. 太子审批     │  最终决策权
│  (Taizi)        │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  4. 执行        │  分配给Trader执行
│  (Execution)    │
└─────────────────┘
```

### 三、需要审批的参数范围

| 参数类型 | 是否需要审批 | 审批级别 |
|---------|-------------|---------|
| 杠杆倍数 > 3x | 必须审批 | 太子 |
| 止损比例修改 > 50% | 必须审批 | 风控+太子 |
| 最大持仓比例 > 30% | 必须审批 | 风控 |
| 新策略启用 | 必须审批 | 风控+太子 |
| 止盈比例调整 > 30% | 必须审批 | 风控 |
| 加仓规则变更 | 必须审批 | 风控+太子 |

### 四、审批标准

**风控审查要点：**
1. 单笔风险是否超过总资金的2%
2. 最大持仓是否超过总资金的30%
3. 杠杆是否在合理范围（≤3x）
4. 止损/止盈比例是否匹配策略类型

**太子审批要点：**
1. 策略是否符合整体投资目标
2. 风险收益比是否合理
3. 策略逻辑是否经过验证

## 学习心得

策略文档是交易系统的"宪法"，审批流程是防止系统崩溃的"安全阀"。两者结合，确保每一笔交易都有据可依、有章可循。
