## 一，核心定义

**确认偏差**：系统性地搜索、解读、记忆支持既有信念的信息，忽略或贬低相反证据。

**通俗解释**：戴了一副只过滤利好新闻的眼镜。

---

## 二，交易中的典型症状

| 症状 | 表现 | 危害 |
|------|------|------|
| 持仓刷利好 | 只关注利好，自动过滤利空 | 错过预警 |
| "研究"变确认仪式 | 先有结论，再找支持证据 | 概率失真 |
| 亏损后合理化 | 找外部原因，不审视决策 | 错误不改正 |

## 三，我的案例

**观察：某Layer2 Token期间**
- 看到唱多报告后立即收藏分享
- 看到唱空报告后仅停留3秒即滑过
- 该Token后下跌40%+，止损决策晚了整整两周

## 四，系统化规避方法

### 1. 强制反向研究（Red Team）

```
每次买入或加仓前，强制要求：
- 撰写"做空理由清单"（字数≥买入理由50%）
- 列出至少3条客观可查的做空依据
- 说明自己如何回应
不写完不操作
```

### 2. 信息源双频道隔离

```
频道1：支持性信息（关注）
频道2：反对性信息（必须追踪）
- 做空机构报告
- 批评性社区帖子
- 负面链上信号
```

### 3. 决策日志双栏记录

```
持仓日志中，同时记录：
- 支持持仓的信号（客观事实）
- 不支持持仓的信号（同样要求客观）
每月底复盘两栏比例
若支持栏长期>>反对栏 → 确认偏差已主导
```

### 4. 魔鬼代言人

```
问自己：如果我最信任的分析师看空，他会怎么说？
指定一个反向声音，必须在重要决策时给出反对意见
```
