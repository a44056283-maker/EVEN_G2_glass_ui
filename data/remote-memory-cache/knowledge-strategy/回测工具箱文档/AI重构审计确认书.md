## 二、交易逻辑MD（爸要求逐条确认）

### 2.1 天眼AI职责边界（修订）

**核心原则**: 天眼AI = V6.5规则的执行者，不是决策者

#### 天眼AI保留的权利

| 动作 | 条件 | 前提 |
|------|------|------|
| 观察报告 | 任何时候 | 随时 |
| 建议半仓 | 任意 | 发飞书群，等爸审核 |
| 强制全平 | score < 40 **AND** 双验证失败 | 冷却30分钟 |
| 上报异常 | V6.5规则被违反 | 发飞书群 |

#### 天眼AI移除的权利
- ❌ 半仓执行权（改为建议）
- ❌ DCA补仓权（V6.5自动驾驶已处理）
- ❌ 低于35分立即平仓（改为40分+双验证）

#### 双验证平仓条件（必须同时满足）

**方向错误 → 全平：**
```
条件A: score < 40
条件B: (做多不在支撑 ±1.5%) OR (做空不在压力 ±1.5%)
条件C: 多时线确认（15m + 30m + 1h 三者至少2个方向一致）

触发: A AND B AND C → 全平
```

**过早入场 → 全平：**
```
条件A: score < 40
条件B: (入场量比 < 对应倍数) OR (做多无净流入) OR (做空无净流出)
条件C: 多时线确认（同上）

触发: A AND B AND C → 全平
```

#### 天眼AI平仓冷却
```
全平后冷却: 30分钟（同方向不可再次干预）
观察报告: 无冷却，持续3小时追踪
建议半仓: 无执行权
```

#### 3小时持续监控（UI要求）
```
入场信号 → 天眼AI进入追踪模式
追踪时长: 3小时
显示内容: 实时评分 + 支撑/压力位距离 + 量比 + 资金流
退出条件: 手动停止 OR 爸审核 OR 3小时到达
```

### 2.2 出山AI职责边界（修订）

**核心原则**: 出山AI = V6.5出场规则审计员 + 资金流反转猎手

#### 出山AI保留的功能

| 功能 | 触发条件 | 动作 |
|------|----------|------|
| V6.5出场规则审计 | 持仓>3h + R2<60 | 报告违规 |
| 资金流反转快信号 | 支撑区放量+净流入 | 猎杀做多 |
| 资金流反转快信号 | 压力区放量+净流出 | 猎杀做空 |
| 巨量反转全平 | 见2.2.2详细 | 全平+反手 |

#### 出山AI移除的功能
- ❌ 半仓出场执行权
- ❌ 自定义止盈建议

#### 2.2.1 资金流反转猎杀（做多）

**触发条件（全部满足）：**
```
A. 支撑区定义: sum(0.990*min(lows_24h) <= l <= 1.010*min(lows_24h)) ≥ 触底确认
B. 放量反抽: vol_ratio ≥ 2.0x
C. 净流入确认: net_flow > 0（做多方向）
D. near_btm收紧: 0.990*low_24h <= l <= 1.010*low_24h
```

**执行**: 直接做多（不需爸审核，因为这是V6.5 L1入场的反向猎杀版本）

#### 2.2.2 巨量反转全平+反手

**做多→做空（压力位巨量净流出）：**
```
条件1: 持仓做多，且到达压力位附近（±3%）
条件2: 连续3根15m K线（或30m+1h叠加）出现巨量净流出
条件3: net_flow < -1.0（做空方向）
触发: 全平做多 + 立即做空
```

**做空→做多（支撑位巨量净流入）：**
```
条件1: 持仓做空，且到达支撑位附近（±3%）
条件2: 连续3根15m K线（或30m+1h叠加）出现巨量净流入
条件3: net_flow > 1.0（做多方向）
触发: 全平做空 + 立即做多
```

**多时线确认计算：**
```
15m × 3根 = 45分钟
或 30m × 2根 + 1h × 1根叠加
确认标准: 至少3个单位时间（15m等效）同向确认
```

### 2.3 V6.5规则绝对优先原则

```

优先级（高→低）:

3. 天眼AI全平（需双验证）
```
