## 一、整体架构

```
父页面 (backtest.html)
└── <iframe id="frame-compare" data-src="tabs/compare.html">
    └── compare.html  ← 完全独立的 JS/CSS/HTML 子页面
        ├── 静态 HTML 骨架（botBar / actionBar / statsRow / content）
        ├── 静态 CSS 样式（CSS 变量 + 组件样式）
        └── JS：状态管理 + API 调用 + 动态渲染
```

**核心原则**：
- 页面加载时：**只渲染静态 HTML 骨架**，Bot 卡片初始为空
- 用户交互时：**分布式按需加载**——点击「⚡ 分析对比」才调用 API
- 数据更新时：**只替换数据区域**，骨架和布局不变

## 六、话术体系（V6.5 / V6.6.1 完整规则）

### V6.5 入场规则
- **L1 量比**：需量比 ≥ 2.5x，做空需 ≥ 2.5x + 下跌趋势
- **L2 流向**：需连续 1 根 K 线同向确认，做多需净流入，做空需净流出
- **L4 S/R**：触底/触顶 ≥ 1 次，距 S/R ±2% 范围

### V6.6.1 动态止盈公式（2026-04-10）
- **触发涨幅** = BASE × (杠杆/10) — 杠杆越高越快触发
- **真实涨幅** = BASE ÷ 10 — 各杠杆止盈到同一价格
- **P1**：基准 15%，卖出 50% 固定，10x→触发 15%，5x→触发 7.5%，实得 +1.5%
- **P2**：基准 25%，卖出 60%×(10/杠杆)，10x→60%，5x→120%→100%，实得 +2.5%
- **P3**：基准 35%，100% 全平防尾仓，实得 +3.5%

### V6.5 出场规则
- **L1 强平**（28 定律）：10X 亏 80% 强平 / 8X 亏 60% / 5X 亏 40%
- **L2 时线止盈**：30m(30%+2x量) / 1h(30%+2x量) / 4h(80%+2.5x量) / 1d(100%+3x量)
- **L3 S/R**（已禁用）：价格距 S/R ±3% 触发，容忍 ±0.5%

### 话术分 4 档
| 档位 | 触发条件 | 话术风格 |
|------|---------|---------|
| 🏆 综合冠军 | score ≥ 80 | 完整展示 L1/L2/L4 + P1/P2/P3 + L1/L2/L3，含 V6.6.1 公式 |
| ✅ 优质端口 | score 60-79 | 展示入场 + 止盈数据，指出薄弱层级 |
| ⚠️ 待优化 | score 40-59 | 展示问题诊断，列出 L1/L2/L4 具体缺陷 |
| ❌ 高危端口 | score < 40 | 红色警告，列出所有不合格项，建议暂停 |

## 七、JS 状态管理

```javascript
let _apiPorts = [];       // API 返回的 ports[]
let _apiSummary = {};     // API 返回的 summary
```

## 八、待优化项

### 8.1 静态 HTML 骨架（最重要）
以下组件需要从 JS 动态生成改为静态 HTML 骨架：
1. **排行榜** `.rank-card`：挂载 8 个 `.rank-row` 骨架，JS 只更新数据和样式
2. **三栏对比** `.tri-grid`：挂载 8 行数据行骨架
3. **盈亏柱图** `.bar-chart`：挂载 8 个 `.bc-row` 骨架
4. **端口详细卡片** `.port-cards`：挂载 8 个 `.port-card` 骨架

### 8.2 布局对齐
需要对照 `live-eval.html` 的布局规范，特别是：
- Bot 卡片样式（border/background/间距）
- 动作栏按钮顺序和样式
- 统计栏格数和颜色
- 卡片展开/折叠交互

### 8.3 话术优化
- Tier 1（冠军）话术太长，需要精简
- P1/P2/P3 数字展示需要格式化，避免连拼（如 `15%×1=15%` 应为 `15%×1=15.0%`）
- 各杠杆预览表（P2 卖出%随杠杆变化）可考虑单独展示
