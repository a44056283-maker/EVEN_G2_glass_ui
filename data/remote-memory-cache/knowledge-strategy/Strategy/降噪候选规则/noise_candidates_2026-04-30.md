# 每日学习采集降噪铁律

> 生效日期: 2026-04-30
> 目标: 所有每日学习、GitHub采集、子代理知识库整合,必须服务于减少噪音下单和提高入场准确率。

## 一、最高目标

每日采集和学习不是为了堆知识,而是为了形成可验证的交易改进假设。

优先级:

1. 减少噪音下单。
2. 提高 V6.5 入场方向准确率。
3. 防止旧规则污染。
4. 提炼可进入 L5 影子实验室的候选规则。
5. 优化天眼/出山解释逻辑,但不得让 AI 越权开仓或平仓。

## 二、每日只采集五类内容

### 1. 合约多空策略

必须同时支持做多和做空。只做多、现货专用、网格现货套利类内容默认降权。

### 2. 资金流/OI/爆仓/盘口

重点采集:

- M1 资金流。
- OI 扩张/收缩。
- 多空爆仓方向。
- 盘口买卖倾斜。
- 点差和滑点风险。

### 3. 支撑压力与假突破识别

重点采集:

- 压力做多风险。
- 支撑做空风险。
- 无量突破。
- 缩量回踩。
- Spring / UT / SOW / SOS。

### 4. 仓位管理和风控

重点采集:

- 5x-10x 合约杠杆风险。
- 量能强弱对应仓位。
- ATR波动异常。
- 过早出场和过度止损风险。

### 5. 历史亏损单归因与噪音过滤

重点采集:

- 反向开单。
- 假突破。
- 低质量放量。
- M1-M5冲突。
- AI过早平仓。

## 三、禁止进入正式规则的内容

以下内容只能记录为黑名单或冲突项,不得进入 V6.5 正式规则:

- FOttStrategy 原生策略。
- 只能做多的策略。
- 现货逻辑直接套合约。
- 不考虑手续费、滑点、资金费率的策略。
- 靠单一指标开仓的策略。
- 会过早平仓的出场规则。
- 与 V6.5 支撑/压力、多时线、多所交叉、M1-M5数据流冲突的规则。

## 四、每日输出必须包含

每个每日学习或采集结果必须输出以下五项:

1. `降噪价值`: 这条知识能减少哪类噪音。
2. `V6.5兼容性`: 是否与现有完整版规则冲突。
3. `多空适配`: 是否同时适用于做多和做空。
4. `候选规则`: 可转成影子实验的具体条件。
5. `禁止项`: 哪些部分不能接入实盘。

## 五、接入流程

所有新知识必须走:

`采集/学习 -> 降噪摘要 -> 候选规则 -> L5影子实验室 -> 7天/300样本 -> 人工审计 -> V6.5正式规则`

不得跳过影子实验直接修改 9090-9097 或 8081-8084 机器人实盘参数。
