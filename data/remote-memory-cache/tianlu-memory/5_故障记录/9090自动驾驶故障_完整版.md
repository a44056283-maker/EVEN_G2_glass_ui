# 9090自动驾驶扫描故障修复记录

**日期**: 2026-04-22
**问题**: 9090自动驾驶扫描不工作，`run_count`始终为0
**影响**: Gate-17656685222 Bot (9090) 无法自动扫描交易对

---

## 症状

```

{
  "enabled": true,
```

---

## 根本原因

**文件**: `api_hunter.py`

**问题函数**: `_load_hunter_scanner()`

```python
# 错误代码（修复前）
freqtrade_root = os.path.dirname(os.path.dirname(os.path.dirname(os.path.dirname(__file__))))
```

**原因**: Python导入`freqtrade`包时，venv中的`__file__`指向：
```
/Users/luxiangnan/freqtrade/.venv311/lib/python3.11/site-packages/freqtrade/rpc/api_server/api_hunter.py
```

计算后`freqtrade_root = .venv311/lib/python3.11/site-packages/`

拼接后路径变成：
```
/Users/luxiangnan/freqtrade/.venv311/lib/python3.11/site-packages/user_data_gate17656685222/strategies
```

该路径不存在，导致`candidates`列表为空，`HunterEyeScanner`加载失败。

---

## 修复方案

**核心思路**: 使用环境变量`FREQTRADE_ROOT`指定正确的freqtrade根目录，不依赖`__file__`推导。

**修复文件**（两处都要改）:
1. `/Users/luxiangnan/freqtrade/freqtrade/rpc/api_server/api_hunter.py`（源码目录）
2. `/Users/luxiangnan/freqtrade/.venv311/lib/python3.11/site-packages/freqtrade/rpc/api_server/api_hunter.py`（venv site-packages）

**修复代码**:
```python
def _load_hunter_scanner():
    """
    动态加载 HunterEyeScanner，自动检测 OKX/Gate user_data 路径。
    """
    import sys
    import os

    # 使用绝对路径（venv中__file__会指向site-packages，需修正）
    freqtrade_root = os.environ.get(
        "FREQTRADE_ROOT",
        os.path.join(os.path.expanduser("~"), "freqtrade")
    )
    # 尝试多个可能的路径（OKX优先）
    candidates = [
        os.path.join(freqtrade_root, 'user_data_okx_9094', 'strategies'),
        os.path.join(freqtrade_root, 'user_data_okx_9095', 'strategies'),
        os.path.join(freqtrade_root, 'user_data_okx_9096', 'strategies'),
        os.path.join(freqtrade_root, 'user_data_okx_9097', 'strategies'),
        os.path.join(freqtrade_root, 'user_data_okx', 'strategies'),
        os.path.join(freqtrade_root, 'user_data_gate17656685222', 'strategies'),
        # Mac B 路径
        os.path.join(freqtrade_root, 'user_data_gate_a63904550', 'strategies'),
        os.path.join(freqtrade_root, 'user_data_gate_a15637798222', 'strategies'),
        os.path.join(freqtrade_root, 'user_data_gate_b15637798222', 'strategies'),
        os.path.join(freqtrade_root, 'user_data_gate_c15637798222', 'strategies'),
    ]
```

**启动命令**需增加环境变量:

## 重要教训

**Python venv site-packages 与源码目录分离**:
- Freqtrade源码: `/Users/luxiangnan/freqtrade/freqtrade/`
- venv安装: `/Users/luxiangnan/freqtrade/.venv311/lib/python3.11/site-packages/freqtrade/`
- Python导入时优先使用venv site-packages
- **修改源码后必须同步更新venv中的文件**，或者用`pip install -e .`保持同步
- 本次因使用了`PYTHONPATH`优先策略，导致源码改了但venv中没同步

**建议**: 以后修改`api_*.py`后，同时更新venv对应文件，或使用`pip install -e .`同步。
