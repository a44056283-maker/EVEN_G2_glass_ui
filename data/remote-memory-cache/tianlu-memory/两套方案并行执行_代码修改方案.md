## 二、两套方案并行执行架构

### 入场方案A：规则性自动驾驶（V6.5本地）

**执行方**: 机器人本地 `check_entry_rules()`
**触发条件**:
- L1量比 ≥ 5.0x（平时）/ 6.0x（周末）
- L2资金流方向与开仓方向一致
- L4 S/R支撑/压力位确认（触底/触顶≥3次，±1%内）

### 入场方案B：M1-M4综合AI评测（天眼AI）

**执行方**: console_server `_entry_scan_once()` → 天眼AI → MiniMax API
**触发条件**:
- verdict = EXECUTE_LONG/SHORT
- confidence ≥ 50%（修改后）
- 轮次确认 ≥ 2轮
- M1/M2/M3数据有效

**英雄卡分值作用**: 作为天眼AI的M1-M4综合输入，AI根据英雄卡数据返回verdict和confidence

### 出场方案A：规则性自动驾驶（V6.5本地）

**执行方**: 机器人本地 `v65_autopilot.py` → `_check_profit_exit()`

## 三、M1-M4综合AI评测执行标准

### 3.1 入场（M1-M4 → 天眼AI → verdict + confidence）

**AI输入（M1）**:
```
15m: ratio, netflow, signal
1h:  ratio, netflow, signal
4h:  ratio, netflow, signal
```
评分规则（M1达到以下任意一条）：
- 三线同向（15m+1h+4h量比同向净流入/流出） → confidence加成
- 双线达标 + S/R确认 → confidence加成
- 量比≥5x + 净流入 → 基础confidence

**AI输入（M2）**:
```
支撑位, 阻力位, 双所/三所验证
```
评分规则：
- 双所/三所验证 → 方向确认置信
- 价格在S/R ±1%内 → S/R位置确认

**AI输入（M3）**:
```
HIGH级巨量K线数量, GIANT_BULL/BEAR, 反转猎杀
```
评分规则：
- HIGH≥3 → 高置信度信号
- 反转猎杀存在 → OBSERVE（出山处理）

**AI输入（M4）**:
```
RSI(15m/1h/4h), ATR%, OI比率
```
评分规则：
- RSI超买(>70) + 做多 → 降低confidence
- RSI超卖(<30) + 做空 → 降低confidence
- OI加仓(≥1.15) → 顺势加分
- ATR高波动(>5%) → 适当降低杠杆

**天眼AI verdict → 执行映射**:
```
EXECUTE_LONG + confidence≥50% + 2轮 → 执行做多
EXECUTE_SHORT + confidence≥50% + 2轮 → 执行做空
FORBIDDEN → 禁止入场
OBSERVE → 观察，出山处理
```

### 3.2 出场（M1-M4 → 出山AI → action + score）

**出山AI评分总分（0-100）**:
- 动态止盈总分 = M1量比分 + M2 S/R分 + M3巨量分 + M4技术分
- 各项满分25分，合计100分

**分项评分（M1）**:
```
量比评分:
  ≥5x: 25分（满分）
  ≥3x: 20分
  ≥2x: 15分
  <2x: 0分

净流入方向:
  做多+净流入: +25分
  做空+净流出: +25分
  方向不符: -25分
```

**分项评分（M2）**:
```
S/R位置:
  价格在支撑±1%内(做多): +25分
  价格在压力±1%内(做空): +25分
  触底/触顶≥3次: +10分
  双所验证: +10分
  三所验证: +15分
```

**分项评分（M3）**:
```
HIGH级巨量K线:
  HIGH≥3: +25分（反转猎杀）
  HIGH=2: +15分
  HIGH=1: +10分
GIANT同向: +10分
```

**分项评分（M4）**:
```
RSI配合:
  做多+RSI<50: +15分
  做多+RSI>70: -20分
  做空+RSI>50: +15分
  做空+RSI<30: -20分
OI加仓(≥1.15): +10分
ATR波动>5%: -10分
```

**总分 → 执行动作**:
```
总分 ≥ 80: EXIT_HALF 50%    # 超越P3，留仓等更高位
总分 ≥ 60: EXIT_HALF 60%  # 超越P2
总分 ≥ 50: EXIT_HALF 50%  # 达到AI执行门槛
总分 ≥ 40: EXIT_HALF 40%  # 维持标准档位
总分 ≥ 20: EXIT_HALF 30%  # 提前至P1
总分 < 20: EXIT_FULL 100% # 趋势衰竭
总分 < 0:  HUNT_REVERSE    # 反转信号，全平+反手
```

## 四、执行流程图

### 入场并行执行

```
每2分钟扫描
    ↓
┌─────────────────┐
│  方案A:规则自动驾驶  │  机器人本地check_entry_rules
│  L1量比≥5x        │
│  L2资金流确认      │  ← 自动驾驶，不依赖AI
│  L4 S/R确认       │
└────────┬─────────┘
         │ 机器人本地执行
         ↓
┌─────────────────┐
│  方案B:M1-M4 AI评测 │  天眼AI综合评测
│  M1量比+M2 S/R   │
│  M3巨量+M4 RSI   │  ← 英雄卡数据输入AI
└────────┬─────────┘
         │ verdict + confidence≥50% + 2轮
         ↓
    执行入场
```

### 出场并行执行

```
每5分钟扫描持仓
    ↓
┌─────────────────┐
│  方案A:规则自动驾驶  │  V6.5本地check_profit_exit
│  P1/P2/P3止盈    │
│  ATR止损         │  ← 机器人本地执行
└────────┬─────────┘
         │ P1≥15% P2≥25% P3≥35%
         ↓
┌─────────────────┐
│  方案B:M1-M4 AI评测 │  出山AI综合评测
│  M1量比+M2 S/R   │
│  M3巨量+M4 RSI   │  ← 英雄卡数据输入AI
└────────┬─────────┘
         │ score阈值 + near_sr条件
         ↓
    执行出场（按score分档）
```
