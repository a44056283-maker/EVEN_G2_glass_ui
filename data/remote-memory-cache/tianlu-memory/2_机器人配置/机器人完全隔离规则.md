# 交易机器人完全隔离规则

## 核心原则

**每个交易机器人必须有独立文件夹和完整隔离，不允许任何共享文件。**

## 必须隔离的内容

每个机器人必须独立拥有：

| 类别 | 具体文件 |
|------|---------|
| **数据库** | `tradesv3_*.sqlite`（含 -wal, -shm）|
| **配置文件** | `config_*_overlay.json` |
| **策略文件** | `FOttStrategy.py`, `SupportResistanceEngine.py` |
| **缓存数据** | `data/`（K线数据），`logs/` |
| **回测结果** | `backtest_results/` |

## 可共享的文件（安全）

| 文件 | 说明 |
|------|------|
| `config_shared.json` | 无API keys，只有共享设置 |
| Freqtrade主体 | `.venv/` 中的 venv 本身 |
| `bot_manager.sh` | 启动脚本（配置表指定不同 userdir 即可）|

## 实践要求

### 1. Overlay Config 必须独立
```json
// 每个端口的 config_XXXX_overlay.json 必须有独立：
//   - db_url（独立SQLite路径）
//   - user_data_dir（独立目录）
//   - exchange API keys（独立账号）
```

### 2. 启动命令必须传递 --userdir
```bash
# bot_manager.sh 中每个端口必须显式指定：
CFG_USERDIR[9090]="/path/to/user_data_xxx"   # 绝对路径，禁止为空
# 启动时传递：
--userdir ${CFG_USERDIR[$port]}              # 禁止省略
```

### 3. userdir 路径必须用绝对路径
- 禁止用相对路径（如 `user_data_okx`）
- 相对路径会被 overlay config 覆盖导致路径分裂
- 正确：`/Users/luxiangnan/freqtrade/user_data_okx_9093`
- 错误：`user_data_okx_9093` 或空字符串

## 验证方法

```bash
# 1. 检查 WAL 活跃（每个端口应有独立 WAL > 0）
ls -lh ~/freqtrade/user_data_xxx/tradesv3*.sqlite-wal

# 2. 检查无重复进程
./bot_manager.sh status

# 3. 检查启动命令中有 --userdir 且为绝对路径
ps aux | grep "freqtrade trade" | grep "9090"
```
