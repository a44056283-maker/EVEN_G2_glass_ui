## 二、天眼AI 入场触发标准（英雄卡驱动）

### 2.1 核心评分体系

天眼AI分析后返回 verdict 和 confidence：

| verdict | confidence条件 | 动作 |
|---------|--------------|------|
| **EXECUTE_LONG** | ≥60%（强制门槛）| 可执行 |
| **EXECUTE_SHORT** | ≥60%（强制门槛）| 可执行 |
| **FORBIDDEN** | — | 禁止入场 |
| **OBSERVE** | — | 观察，不执行 |

**英雄卡展示**：confidence_scaled = confidence_raw × 100（0-100分）

### 2.2 入场执行必须同时满足的条件

```
① verdict = EXECUTE_LONG/SHORT
② confidence ≥ 60%（硬门槛，即使AI返回EXECUTE，低于60%也降级为FORBIDDEN）
③ 轮次确认 ≥ 2轮（连续2轮同向信号才执行）
④ V6.5规则预过滤通过
⑤ M1数据有效（ratio > 0.05）
```

### 2.3 多时线共振（V6.5参考值，非硬门槛）

| 时线 | 触发阈值 | 做多条件 | 做空条件 |
|------|---------|---------|---------|
| 15m | ≥5.0x | ratio≥5.0 + netflow>0 | ratio≥5.0 + netflow<0 |
| 1h | ≥3.0x | ratio≥3.0 + netflow>0 | ratio≥3.0 + netflow<0 |
| 4h | ≥2.0x | ratio≥2.0 + netflow>0 | ratio≥2.0 + netflow<0 |

**多时线共振规则**：
- 15m+1h+4h三线同向 = 高置信度信号
- 活的M1-M3数据流强烈共振 → 即使V6.5未触发硬共振，也可EXECUTE
- V6.5触发但活的M1-M3背离 → FORBIDDEN
- 无量+背离 = FORBIDDEN（无论V6.5是否触发）

### 2.4 M4 RSI修正规则

| 场景 | 动作 |
|------|------|
| M4 RSI >70 + 入场做多 | 降低做多置信度 |
| M4 RSI <30 + 入场做空 | 降低做空置信度 |
| M4 OI加仓（≥1.15）| 顺势确认，大加分 |
| M4 OI减仓（≤0.85）| 警惕信号 |
| M4 ATR >5% | 适当降低杠杆 |

## 三、出山AI 出场触发标准（英雄卡驱动）

### 3.1 核心评分体系

**英雄卡展示**：`🚨反转猎杀` + score + reversal_verify详情

→ 出山AI只记录，不重复执行

#### 条件3：紧贴S/R位 + 浮盈达标（出山AI主动执行）

#### 条件5：时线止盈（飞书通知，V6.5执行）

## 六、英雄卡当前问题与修复方向

### 问题1：英雄卡数据是摆设
**根因**：英雄卡数据已采集并展示，但未与执行逻辑挂钩

**修复方向**：
1. 英雄卡评分直接映射到执行阈值
2. 当 M1量比+ M2 S/R + M3巨量 同时达标时，自动触发对应动作

### 问题2：天眼AI入场 verdict 未被英雄卡直接引用
**根因**：英雄卡显示 M1/M2/M3 分值，但天眼AI独立调用MiniMax API

**修复方向**：英雄卡评分达到以下任意组合时，直接触发：
- M1量比 ≥5x + M2 S/R方向匹配 + M3 HIGH≥3 → 强信号
- M1三线共振 + M2双所验证 → 高置信度入场

### 问题3：出山AI出场依赖AI评测结果
**根因**：出山AI依赖MiniMax API返回的action，无法直接根据英雄卡分值执行

**修复方向**：增加规则化执行层（不依赖AI API）：
- 英雄卡分值达标 → 规则直接触发出场
- AI评测作为补充置信度验证
