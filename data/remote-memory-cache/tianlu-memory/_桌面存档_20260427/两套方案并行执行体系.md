## 三、入场体系

### 3.1 方案A：规则性自动驾驶（V6.5本地）

**执行方**: 机器人本地 `check_entry_rules()`

| 层级 | 检查项 | 条件 | 说明 |
|------|--------|------|------|
| L1 | 量比过滤 | **≥5.0x**（平时）/ **≥6.0x**（周末）| 不是2.5x！是5.0x |
| L1 | 周末升级 | 周五17:00后 + 周六日 | 量比自动升为6.0x |
| L1 | 做多规则 | 量比≥5.0x + 趋势上涨 + **在支撑线下方** | 距支撑≤1% |
| L1 | 做空规则 | 收盘价<基线 + 量比≥7.0x + 下跌趋势 | |
| L2 | 资金流 | 方向与开仓一致（做多=LONG/净流入，做空=SHORT/净流出）| 或 |
| L2 | K线确认 | 连续N根K线同向（默认1根）| |
| L4 | S/R支撑 | 支撑位存在 + 触底≥3次 + 价格距支撑±1%内 | 做多 |
| L4 | S/R压力 | 压力位存在 + 触顶≥3次 + 价格距压力±1%内 | 做空 |
| - | 冷却 | 止盈出场后同向禁止入场 | 冷却时间内禁止 |

**执行**: 机器人本地满足全部条件后自动入场，**不依赖AI**

---

### 3.2 方案B：M1-M4综合AI评测（天眼AI）

**执行方**: console_server `_entry_scan_once()` → 天眼AI → MiniMax API

#### AI输入数据源

| 数据 | 来源 | 用途 |
|------|------|------|
| M1资金流 | 英雄卡 | 15m/1h/4h量比+netflow+signal |
| M2 S/R | 英雄卡 | 支撑/压力位+双所/三所验证 |
| M3巨量K线 | 英雄卡 | HIGH/GIANT_BULL/GIANT_BEAR/反转猎杀 |
| M4技术指标 | 英雄卡 | RSI/ATR/OI |

#### 天眼AI verdict体系

| verdict | 含义 | 执行? |
|---------|------|-------|
| EXECUTE_LONG | 执行做多 | ✅ 需满足置信度门槛 |
| EXECUTE_SHORT | 执行做空 | ✅ 需满足置信度门槛 |
| FORBIDDEN | 禁止入场 | ❌ 不执行 |
| OBSERVE | 观察 | ⚠️ 不执行，等待进一步信号 |

#### 置信度门槛（爸确认: 50%）

```
EXECUTE + confidence ≥ 50% + 2轮确认 → 执行入场
EXECUTE + confidence < 50% → FORBIDDEN（降级）
FORBIDDEN → 不执行
OBSERVE → 不执行（观察）
```

#### 英雄卡 → AI评分映射（M1）

| 条件 | 基础置信度 |
|------|-----------|
| 15m+1h+4h三线同向 + 量比≥5x/3x/2x | 90% |
| 15m+1h双线达标 + S/R确认 | 80% |
| 15m单线达标 + 量比≥5x | 70% |
| M1量比<2x | 40% |
| 无量+背离 | FORBIDDEN |

#### 置信度修正（M4调整）

| 条件 | 调整 |
|------|------|
| M4 RSI>70 + 做多 | 置信度 -20% |
| M4 RSI<30 + 做空 | 置信度 -20% |
| M4 OI加仓(≥1.15) | 置信度 +10% |
| M4 ATR>5% | 适当降低杠杆 |
| 三所同向(Gate+OKX+BNB) | 置信度 +15% |

#### 轮次确认

```
第1轮: verdict=EXECUTE + conf≥50% → 记录，不执行
第2轮: verdict=EXECUTE + conf≥50% + 同向 → 执行入场
第3轮+: verdict变化 → 重置轮次，从第1轮开始
超时30分钟 → 轮次重置
```

| 条件 | 得分 |

| 条件 | 得分 |
|------|------|
| 价格紧贴S/R位(±1%) + 方向匹配 | 25分 |
| 触底/触顶≥3次 | +10分 |
| 双所验证 | +10分 |
| 三所验证 | +15分 |
| 远离S/R位 | 0分 |

##### M3评分（巨量K线）

| 条件 | 得分 |
|------|------|
| HIGH≥3（反转猎杀）| 25分 |
| HIGH=2 | 15分 |
| HIGH=1 | 10分 |
| GIANT同向 | +10分 |
| 无反转猎杀 | 基准分 |

##### M4评分（技术指标）

| 条件 | 得分 |
|------|------|
| RSI配合方向 | +15分 |
| RSI极端背离方向 | -20分 |
| OI加仓(≥1.15) | +10分 |
| ATR高波动(>5%) | -10分 |

| 总分 | verdict | 卖出比例 | 说明 |

## 五、并行执行原则

### 5.1 两套方案关系

```
方案A（规则自动驾驶）和 方案B（AI综合评测）始终并行运行

方案A触发 → 机器人本地执行（独立于AI）
方案B触发 → console_server执行（依赖AI verdict/score）

两套方案互不干扰，各自独立判断和执行
```

### 5.2 防重复执行

```
方案A全平（P3=100%）→ 持仓消失 → 方案B不再评测该持仓
方案B先执行50%止盈 → 仍有50%持仓 → 方案A继续监控剩余仓位
```
